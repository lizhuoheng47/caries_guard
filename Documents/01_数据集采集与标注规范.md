# 数据集采集与标注规范

本文档不再把“训练数据治理平台”写成当前已实现事实，而是按当前代码边界说明：哪些数据来源已经在业务平台中存在，哪些采集、脱敏、标注、快照、审批流程仍属于治理规范层。

## 1. 文档边界

当前 `backend-java` 仓库已实现的是业务平台，不是训练平台。

因此本文档分两层：
- 当前业务平台已经承载的数据来源
- 面向训练数据治理的建议流程

以下内容当前未在本仓库中落地为独立平台：
- 标注平台
- 训练数据快照平台
- Dataset Card 平台
- Gold Set 平台
- 模型审批上线平台

## 2. 当前业务平台可提供的数据来源

### 2.1 图像与附件来源

| 表/对象 | 当前作用 |

| `med_image_quality_check` | 图像质检结果 |
| 本地文件系统对象 | 当前真实图片文件存储位置 |

| 表 | 当前作用 |

| `med_case` | 病例主实体 |

| `med_case_tooth_record` | 牙位级结构化记录 |
| `med_case_status_log` | 病例状态链路证据 |

### 2.3 AI 结果与反馈来源

| 表 | 当前作用 |
| --- | --- |

## 3. 当前不能误写成已实现的能力

以下治理能力在来源稿中是合理设计，但当前没有平台实现：
- 采集端审批流
- 标注任务派发系统
- 标注质检工作台
- 训练样本快照编号系统
- Gold Set 独立维护系统
- Dataset Card 独立产出系统
- 训练导出审批台

因此当前正确说法是：
- 业务平台已经积累了训练候选原始来源
- 训练治理流程需要在业务平台之外或业务平台扩展能力上实现

## 4. 当前代码与数据治理的真实边界

### 4.1 已实现边界

当前业务平台已经做了：
- 患者敏感数据保护
- 图像上传与元数据管理
- 图像质检留痕
- AI 结果回写
- 医生纠偏反馈入库
- 风险结果入库
- `modelVersion` 留痕

### 4.2 未实现边界

当前业务平台尚未做：
- 样本脱敏导出审批
- 训练集快照冻结
- 标注任务管理
- 标注结果复核流程
- Gold Set 独立维护

## 5. 建议的数据采集与标注流程

以下流程可以保留为治理规范，但必须标注为“建议流程”：

1. 业务平台沉淀原始病例与图像
2. 从业务平台筛选候选样本
3. 对候选样本做脱敏导出
4. 进入独立标注流程
5. 医生复核形成高可信标注结果
6. 冻结为数据集快照
7. 形成 Dataset Card 与版本说明
8. 进入离线评估与人工审批流程

## 6. 训练候选样本筛选建议

### 6.1 候选来源优先级

建议优先从以下数据生成训练候选：
1. `ana_correction_feedback`：医生已明确修正过的样本
2. `med_case_tooth_record`：带牙位结构化结果的病例
3. `med_case_diagnosis`：已确认诊断的病例
4. `med_image_quality_check`：质检合格图像
5. `ana_result_summary`：已回写分析摘要的任务

### 6.2 候选样本最小追溯信息

每条候选样本至少应保留：
- `case_id`
- `patient_id` 或脱敏后的 patient surrogate key
- `image_id`
- `attachment_id`
- `task_id`
- `task_no`
- `model_version`
- `source_image_id`（如有牙位级追溯）
- 医生修正记录 ID
- 导出时间与导出批次号

## 7. 脱敏与导出要求

### 7.1 必须脱敏的内容

- 患者姓名
- 手机号
- 证件号
- 明确身份相关文本
- 非必要的联系信息

### 7.2 当前平台已具备的基础

当前系统已具备：
- 敏感字段加密
- 敏感字段哈希
- 敏感字段脱敏显示

### 7.3 当前平台未具备的导出治理

当前系统尚未具备：
- “脱敏导出审批”业务流程
- “导出即登记快照号”机制

建议补强：
- `training_candidate_flag`
- `desensitized_export_flag`
- `dataset_snapshot_no`
- `review_status_code`
- `reviewed_by`
- `reviewed_at`

## 8. 标注规范建议

当前仓库没有标注系统，因此这里只保留推荐规范。

### 8.1 标注对象层级

建议分三层：
1. 图像级：是否可用、图像类型、质量问题
2. 病灶/牙位级：牙位、牙面、问题类型、严重程度、来源图像
3. 病例级：整体风险、是否建议复查、综合说明

### 8.2 质量控制建议

建议保留：
- 双人标注
- 高争议样本医生裁定
- Gold Set 复核集
- 标注版本号
- 质控抽检记录

## 9. Dataset Card 与 Gold Set

来源稿中的这两项应保留，但必须改写为：
- 当前建议治理产物
- 当前未在业务平台中实现独立模块

### 9.1 Dataset Card 应包含

- 数据来源范围
- 数据时间范围
- 采集机构范围
- 图像类型构成
- 患者级划分方式
- 脱敏说明
- 已知偏差与限制
- 对应模型版本和实验用途

### 9.2 Gold Set 应包含

- 高可信样本清单
- 专家裁定过程
- 版本号
- 适用任务
- 不能直接用于训练的限制说明

## 10. 当前最重要的统一结论

1. 当前业务平台已经提供训练候选原始数据来源。
2. 当前业务平台没有实现独立标注平台。
3. 当前业务平台没有实现训练快照治理平台。
4. `ana_correction_feedback` 是当前最接近“训练候选入口”的业务表。
5. 训练治理能力可以保留在文档中，但必须明确为“建议扩展”，不能写成“当前已交付”。
