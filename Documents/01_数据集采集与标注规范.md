# 数据集采集与标注规范

更新日期：2026-04-15

本文档区分“当前 Java 业务平台已经落地的训练数据相关能力”和“来源设计中应保留但尚未实现为独立平台的标注/训练治理能力”。

## 1. 当前结论

1. 当前仓库没有独立标注平台源码。
2. 当前仓库没有独立训练数据快照平台源码。
3. 当前仓库没有独立 ModelAdmin 平台源码。
4. 当前 Java 业务库已经具备训练候选数据的业务来源：影像、AI 结果、医生修正、风险评估、病例牙位追溯。
5. V014 已在 `ana_correction_feedback` 中补入训练准入治理字段，能把“医生修正 -> 候选样本 -> 脱敏导出 -> 数据集快照”的流程落到业务数据层。
6. 当前对象存储默认使用 MinIO，训练数据导出时应使用 `med_attachment.bucket_name`、`med_attachment.object_key`、`med_attachment.storage_provider_code` 和下载 URL 获取影像，不应假设一定是本地路径。

## 2. 当前可作为训练候选来源的数据

| 数据类别 | 当前表 | 用途 |
| --- | --- | --- |
| 影像附件 | `med_attachment` | 原始影像二进制对象元数据，包含 bucket、objectKey、MD5、provider |
| 病例影像 | `med_image_file` | 影像与病例关系、影像类型、牙位范围、主槽位 |
| 影像质检 | `med_image_quality_check` | 影像质量、质检结果、质检意见 |
| 病例 | `med_case` | 病例状态、患者、就诊、机构、主治医生 |
| 牙位记录 | `med_case_tooth_record` | 牙位、诊断、病灶严重程度、`source_image_id` 追溯 |
| AI 任务 | `ana_task_record` | 任务号、模型版本、状态、Trace、推理耗时 |
| AI 摘要 | `ana_result_summary` | 原始结果、最高严重程度、不确定性、建议复核标记 |
| AI 可视化资产 | `ana_visual_asset` | heatmap、mask、overlay 等可解释性资产 |
| 风险评估 | `med_risk_assessment_record` | 风险等级、依据、建议、下次随访建议 |
| 医生修正 | `ana_correction_feedback` | 修正内容、候选标记、审核状态、快照编号 |
| 模型版本 | `ana_model_version_registry` | 模型版本登记与审批状态 |

## 3. 训练候选数据准入规则

`ana_correction_feedback` 当前字段已经支持最小准入治理：

| 字段 | 说明 |
| --- | --- |
| `training_candidate_flag` | 是否进入训练候选池，`1` 表示可以进入候选流程 |
| `desensitized_export_flag` | 是否已经通过脱敏导出 |
| `dataset_snapshot_no` | 所属数据集快照编号 |
| `review_status_code` | 审核状态，默认 `PENDING` |
| `reviewed_by` | 审核人用户 ID |
| `reviewed_at` | 审核时间 |
| `is_exported_for_train` | 旧有训练导出标记，继续保留兼容 |

建议准入流程：

1. 医生提交修正反馈。
2. Java 写入 `training_candidate_flag=1`、`review_status_code=PENDING`。
3. 数据治理人员或后续管理端审核反馈是否可进入训练集。
4. 审核通过后执行脱敏导出，写入 `desensitized_export_flag=1`。
5. 生成数据集快照编号，写入 `dataset_snapshot_no`。
6. 导出清单中保留 `taskNo`、`modelVersion`、`attachmentMd5`、`source_image_id`、`caseId`、`toothCode`，保证可追溯。

## 4. 采集规范

当前业务平台采集影像时应满足：

| 要求 | 当前承载 |
| --- | --- |
| 文件唯一性 | `med_attachment.md5` |
| 对象定位 | `bucket_name` + `object_key` + `storage_provider_code` |
| 访问方式 | `AttachmentAppService.createAccessUrl` 生成短时 URL |
| 存储 provider | `MINIO` 或 `LOCAL_FS` |
| 病例归属 | `med_image_file.case_id` |
| 影像类型 | `med_image_file.image_type_code` |
| 牙位范围 | `med_image_file.tooth_range_json` |
| 主槽位 | `med_image_file.primary_slot` |
| 质量状态 | `med_image_quality_check` |

Python 或训练导出程序不得直接假设 `objectKey` 是本地文件路径。MinIO 场景下应通过短时 URL 或 MinIO SDK 读取。

## 5. 标注规范

当前没有独立标注平台，标注规范保留为后续扩展口径。若后续建设标注平台，建议最小对象包括：

| 标注对象 | 建议字段 |
| --- | --- |
| 图像级标注 | `attachmentId`、`imageId`、`caseId`、`labelVersion`、`annotatorId`、`reviewStatus` |
| 牙位级标注 | `toothCode`、`lesionTypeCode`、`severityCode`、`sourceImageId` |
| 病灶框/分割 | `shapeType`、`pointsJson`、`maskObjectKey`、`confidence` |
| 质控结果 | `reviewerId`、`reviewedAt`、`rejectReason` |
| 数据集归属 | `datasetSnapshotNo`、`splitCode`、`goldSetFlag` |

当前 Java 业务库可先把医生修正反馈作为弱标注/候选标注来源，但不能在文档中写成已完成完整标注平台。

## 6. Dataset Card 建议

每个训练数据快照应记录 Dataset Card。建议字段：

| 字段 | 说明 |
| --- | --- |
| `datasetSnapshotNo` | 快照编号 |
| `sourceSystem` | 来源系统，当前为 Java 业务平台 |
| `sourceModelVersion` | 候选样本来源模型版本 |
| `sampleCount` | 样本数 |
| `caseCount` | 病例数 |
| `positiveRatio` | 阳性比例 |
| `riskLevelDistribution` | 风险等级分布 |
| `severityDistribution` | 龋坏严重程度分布 |
| `goldSetFlag` | 是否包含金标准子集 |
| `desensitizationPolicy` | 脱敏策略 |
| `approvalStatus` | 审批状态 |
| `approvedBy` | 审批人 |
| `approvedAt` | 审批时间 |

## 7. Gold Set 建议

Gold Set 仍属于流程规范，当前没有独立表。建议后续规则：

1. 每个 Gold Set 样本必须有专家复核结论。
2. 每个样本必须保留 `attachmentMd5`，避免二进制漂移。
3. 每个样本必须保留 `modelVersion`，区分来源模型。
4. 每个牙位级标注必须保留 `source_image_id`。
5. Gold Set 不直接被普通训练流程覆盖，需单独审批。

## 8. 脱敏要求

业务库已经对部分敏感字段采用密文、hash、masked 三列模式。训练导出时仍需遵守：

1. 不导出姓名、手机号、证件号、邮箱明文。
2. 使用 `*_masked` 字段展示需要，不把 `*_enc` 交给训练流程。
3. 导出病例和影像时保留匿名化 ID，不保留可逆身份信息。
4. 导出操作应记录操作者、时间、快照编号和审批状态。
5. 训练候选必须优先从 `ana_correction_feedback` 的审核结果筛选。

## 9. 与模型版本治理的关系

V014 新增 `ana_model_version_registry`，当前可记录最小模型治理信息：

| 字段 | 说明 |
| --- | --- |
| `model_code` | 模型编码 |
| `model_version` | 模型版本 |
| `model_type_code` | 模型类型 |
| `approved_flag` | 是否批准上线 |
| `approved_by` | 审批人 |
| `approved_at` | 审批时间 |
| `status` | `CANDIDATE`、`EVALUATING`、`APPROVED`、`RETIRED` |

当前还没有完整模型发布平台，但训练数据导出和看板统计必须保留 `modelVersion`，否则无法定位模型效果变化。
