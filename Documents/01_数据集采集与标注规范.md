# 01 数据集采集与标注规范

## 1. 目标

本规范用于指导影像算法评估数据、RAG 知识库数据和业务反馈数据采集。当前不以通用大模型微调为主要目标，因此采集重点不是 prompt 微调样本，而是高质量知识、结构化病例上下文和可评估的影像标注。

## 2. 数据类型

### 2.1 影像评估数据

用于 quality、detection、segmentation、grading、uncertainty 校准。

必须标注：

- 影像类型；
- 牙位；
- 病灶区域；
- 分级标签 `C0/C1/C2/C3`；
- 标注医生；
- 复核状态；
- 标注版本；
- 图像质量问题。

### 2.2 知识图库数据

用于通用大模型 RAG：

- 指南；
- 诊疗路径；
- 科普材料；
- 分级说明；
- 风险因素；
- 报告模板；
- 复查建议；
- 业务规则。

每条知识必须记录来源、版本、审核人、适用范围和失效时间。

### 2.3 业务反馈数据

用于持续评估：

- AI 分级与医生结论差异；
- high uncertainty 是否正确触发复核；
- 报告修改记录；
- 随访结果；
- RAG 回答评分；
- 医生采纳率。

## 3. 标注流程

```text
数据入库
  -> 脱敏检查
  -> 初标
  -> 复核
  -> 仲裁
  -> 版本冻结
  -> 评估集发布
```

## 4. 分级标准

| 标签 | 含义 |
| --- | --- |
| `C0` | 未见明确龋损 |
| `C1` | 早期或轻度可疑 |
| `C2` | 中度龋损或需重点关注 |
| `C3` | 重度龋损或高度可疑 |

分级标签用于辅助决策，不替代医生诊断。

## 5. Uncertainty 标注

建议记录：

- 标注者一致性；
- 图像质量问题；
- 病灶边界不清；
- 牙位遮挡；
- 分级边界样本；
- 是否需要复核。

系统规则：

```text
uncertaintyScore >= threshold -> needsReview=true
```

## 6. RAG 评估集

RAG 标准问答集应覆盖：

- 龋病基础知识；
- 影像结果解释；
- 分级说明；
- 复查建议；
- 风险因素；
- 患者科普；
- 拒答和安全边界。

评分维度：

- 检索命中；
- 引用准确；
- 医学表达保守；
- 无幻觉；
- 不越权诊断；
- 可读性。

## 7. 合规要求

- 不采集明文身份信息进入 AI 数据集；
- 影像和病例必须脱敏；
- RAG 知识来源必须可授权；
- 评估集变更必须留版本；
- 医生标注和复核操作需可追溯。
