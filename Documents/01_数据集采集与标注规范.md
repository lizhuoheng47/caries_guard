## 一、修订说明

本版规范相对原始版本，重点完成以下修订：

- 单独明确“**业务运行数据**”与“**模型训练数据**”的边界；
- 删除与当前任务关联弱、敏感性较高且难以证明必要性的字段；
- 将目标规模从单一理想化口径，修订为“**竞赛可交付底线 + 理想目标**”双口径；
- 补充数据回流进入训练前的审批机制；
- 强化数据卡片、质控记录、标注一致性等答辩所需证据材料。

## 二、总则

### 2.1 目的

本规范用于指导“多模态龋齿智能识别与分级预警平台”的数据采集、去标识化、标注、质控、版本管理和训练准入流程，确保：

- 数据合规；
- 标注可靠；
- 数据划分无泄漏；
- 模型训练来源清晰；
- 答辩证据链完整。

### 2.2 核心原则

1. **合规优先**：所有数据工作必须遵守法律法规、伦理要求和合作机构规定。
2. **临床导向**：采集与标注标准应贴近真实口腔诊疗与筛查场景。
3. **患者级隔离**：训练、验证、测试严格按患者级划分，禁止同一患者跨集合泄漏。
4. **质量优先**：宁缺毋滥，高质量样本优先于大规模低质样本。
5. **边界清晰**：业务系统运行数据与模型训练数据必须分层治理。
6. **可追溯性**：所有样本、标注、修改、版本和导出都必须可追溯。

### 2.3 名词定义

| 术语 | 定义 |
|---|---|
| 受试者 | 提供口腔影像及相关信息的自然人 |
| 业务运行数据 | 用于平台日常运行的病例、患者、影像、报告、随访等数据 |
| 脱敏副本 | 从业务运行数据中导出的去标识化数据，用于标注和训练准备 |
| 训练数据集 | 经脱敏、质控、版本固化后用于模型训练与评估的数据快照 |
| 一级标注员 | 经培训考核合格的初级标注人员 |
| 二级复核医生 | 负责临床复核的执业口腔医师 |
| 仲裁专家 | 负责争议样本裁定的高级临床专家 |
| Gold Set | 经过多轮复核确认的高置信度样本集合 |
| Dataset Card | 描述数据来源、分布、限制、风险和指标的数据集说明文件 |

## 三、伦理与合规要求

### 3.1 伦理审查

1. 正式采集合作机构数据前，必须获得相应伦理审批或机构许可；
2. 伦理材料至少应包含：研究方案、知情同意模板、数据安全方案、研究成员资质说明；
3. 伦理批件编号或授权编号必须登记在数据集元数据中；
4. 若当前阶段尚未完成真实临床合作采集，答辩时必须明确口径为“公开数据 + 小规模真实数据验证”或“公开数据主导”，不得夸大数据来源。

### 3.2 知情同意

1. 涉及真实患者数据时，需取得受试者或监护人知情同意；
2. 知情同意中应明确：
   - 数据仅用于科研与教学或比赛项目研究；
   - 数据将进行去标识化处理；
   - 受试者可申请撤回；
   - 不影响正常诊疗权益；
3. 知情同意文件与业务影像数据不得同库同路径存放。

### 3.3 去标识化要求

#### 必须删除的直接标识符

- 姓名、身份证号、手机号、住址、病历号；
- DICOM 中的 Patient Name、Patient ID、Institution Name 等直接标识字段；
- 口内照中明显可识别个人身份的标志。

#### 必须脱敏的准标识符

- 出生日期仅保留年龄或出生年份；
- 就诊日期按月记录，不保留具体日；
- 机构名称使用编码替代。

#### 替换策略

- 原始 `patient_id` 使用带项目盐值的哈希转换为 `patient_uuid`；
- 映射表独立保管，不与训练数据集一同分发；
- 脱敏过程保留审计日志。

### 3.4 数据安全要求

- 原始数据应存于受控环境中；
- 脱敏数据与原始业务数据分层存储；
- 所有导出、下载、标注、删除动作需留痕；
- 成员退出项目后应立即撤销访问权限。

## 四、业务数据与训练数据边界

### 4.1 为什么必须单独治理

业务系统中的患者档案、影像、报告和随访记录是为临床辅助流程服务的，其数据结构和权限管理目标，与模型训练所需的数据治理目标不同。若不单独分层，容易引发：

- 合规风险；
- 权限混乱；
- 数据泄漏；
- 训练样本来源不可追踪；
- 答辩时被质疑“直接拿业务库训模型”。

### 4.2 数据分层模型

| 层级 | 数据内容 | 是否包含身份信息 | 用途 |
|---|---|---|---|
| 业务运行层 | 患者档案、原始影像、报告、随访记录 | 是 | 平台运行 |
| 脱敏副本层 | 去标识化影像、结构化特征、病例标签 | 否 | 标注与训练准备 |
| 标注质控层 | 标注结果、复核记录、仲裁结果 | 否 | 训练集构建与评估 |
| 训练数据层 | 固化的数据版本快照 | 否 | 模型训练 |
| 模型资产层 | 模型权重、评估指标、版本说明 | 否 | 推理服务与治理 |

### 4.3 准入规则

- 业务运行数据**不得直接用于训练**；
- 进入训练的数据必须满足：
  1. 完成脱敏；
  2. 完成标注与质控；
  3. 登记版本号；
  4. 具备来源说明；
  5. 通过数据管理员与临床顾问审核。

## 五、数据采集规范

### 5.1 数据来源策略

采用“**公开数据集 + 合作机构数据**”双轨策略。

#### 5.1.1 公开数据集

公开数据用于：

- 模型预训练与基础能力构建；
- 算法早期验证；
- 竞赛阶段的稳定复现实验。

使用公开数据集时必须：

- 核查许可证是否允许学术研究；
- 记录来源与引用信息；
- 区分训练集和展示样例来源。

#### 5.1.2 合作机构数据

合作机构数据用于：

- 真实场景验证；
- 误差分析；
- 跨来源鲁棒性评估；
- 支撑答辩中的应用可信度。

### 5.2 样本规模双口径

为避免文档口径过大但实际无法支撑，采用双口径管理。

#### 竞赛可交付底线

- 患者数：不少于 120 例；
- 图像数：不少于 300 张；
- 至少覆盖 2 种模态；
- 至少具备一部分双人复核样本；
- 至少形成可用的 Gold Set 子集。

#### 理想目标

- 患者数：300 例以上；
- 图像数：600–1000 张；
- 覆盖全景片、根尖片、口内照 3 种模态；
- 来自至少 2 个来源单位；
- 建立较完整的 Gold Set 和外部验证集。

答辩时必须区分“已完成规模”和“目标规模”，不得混用。

### 5.3 纳入与排除标准

#### 纳入标准

- 具备可用口腔影像；
- 签署知情同意或符合公开数据集授权；
- 影像质量满足最低要求；
- 具备基本结构化信息或关键字段不为空。

#### 排除标准

- 大面积金属伪影严重影响判读；
- 影像严重模糊、曝光异常或截断严重；
- 同一患者同一时间点重复影像仅保留最佳样本；
- 无法确认标注范围且不具备仲裁条件的影像。

### 5.4 采集模态与质量要求

#### 全景片

- 建议分辨率不低于 1500×3000；
- 优先保留 DICOM 原始文件；
- 必须包含完整上下牙弓。

#### 根尖片

- 建议分辨率不低于 800×1200；
- 必须包含目标牙齿及其根尖区域。

#### 口内彩照

- 建议分辨率不低于 1920×1080；
- 尽量避免强反光、严重遮挡和大面积模糊；
- 建议采集标准视角或记录拍摄位姿说明。

### 5.5 影像质量初筛

| 检查项 | 基本要求 |
|---|---|
| 清晰度 | 牙冠边缘可辨、关键区域不糊 |
| 曝光 | 无大面积过曝欠曝 |
| 完整性 | 主要目标区域完整 |
| 遮挡 | 无关键区域遮挡 |
| 畸变 | 无明显影响判读的几何变形 |

不合格影像要么重采，要么进入隔离区，不得直接混入正式数据集。

## 六、结构化信息采集规范

### 6.1 字段设计原则

结构化信息仅采集与风险评估和场景解释直接相关、且具有合理临床意义的字段。

**不采集当前任务无明确必要性的敏感属性字段。**

### 6.2 推荐字段

```json
{
  "patient_uuid": "hashed_id",
  "age": 25,
  "gender": "F",
  "brushing_freq_per_day": 2,
  "sugar_diet_score": 3,
  "fluoride_toothpaste_use": true,
  "family_caries_history": false,
  "last_dental_check_months": 12,
  "orthodontic_history": false,
  "previous_caries_count": 2,
  "smoking": false
}
```

### 6.3 缺失值处理

- 缺失值必须显式标记为 `null`；
- 严禁以 `0` 冒充未知值；
- 训练前单独执行缺失值统计和处理策略说明。

### 6.4 字段删除说明

本版规范不再保留原方案中的 `ethnicity_optional` 字段，原因如下：

- 与当前龋病筛查任务的直接建模价值不足；
- 容易引入敏感属性争议；
- 无法在竞赛周期内充分论证其必要性与公平性影响。

## 七、标注协议

### 7.1 标注任务范围

每张影像原则上完成以下 4 类标注：

| 标注类型 | 粒度 | 输出形式 | 作用 |
|---|---|---|---|
| 牙位检测 | 牙级 | BBox + FDI 编号 | 牙位先验、区域定位 |
| 病灶分割 | 像素级 | 二值 Mask | 分割主任务 |
| 龋病分级 | 牙级 | C0/C1/C2/C3 | 分级任务 |
| 病灶类型 | 牙级 | 类别枚举 | 辅助分析 |

### 7.2 牙位检测标注规范

- 严格使用 FDI 双数字法；
- BBox 紧贴目标牙冠区域，不做过大外扩；
- 遮挡严重到无法确认牙位时，可标记为“不可判读”；
- 种植牙、修复体、残根等特殊情况要标记 `tooth_status`。

### 7.3 病灶分割规范

- 仅标注可确认病灶区域；
- 边界不清时应降低置信度并可送仲裁；
- 禁止把不同牙位的病灶连为一个整体；
- 禁止未经人工确认直接采纳自动分割结果。

### 7.4 简化分级规范

项目采用简化四级体系，降低标注复杂度，增强一致性：

| 类别 | 含义 |
|---|---|
| C0 | 无明显龋损 |
| C1 | 早期脱矿或浅表变化 |
| C2 | 中度龋，存在更明确病变迹象 |
| C3 | 深度龋，损害更重 |

### 7.5 标注格式要求

每张影像对应一个 JSON 标注文件，至少应包含：

- `image_id`
- `patient_uuid`
- `image_type`
- `annotation_version`
- `annotator_l1`
- `annotator_l2`
- `quality_flag`
- `difficulty`
- `teeth[]`

建议保留 `confidence` 字段表示标注员自评置信度。

## 八、质量控制流程

### 8.1 三级质控链路

```text
原始影像
  ↓
L1 初级标注
  ↓
L2 医生复核
  ↓
争议样本进入 L3 仲裁
  ↓
正式数据集 / Gold Set
```

### 8.2 L1 标注要求

- 必须经过培训与考核；
- 标注量设置上限，防止疲劳标注；
- 对低置信度样本主动标记；
- 所有操作留痕。

### 8.3 L2 复核要求

复核医生需确认：

- 牙位编号正确性；
- 分割边界合理性；
- 分级与临床判断一致性；
- 是否有遗漏病灶。

复核结论分为：

- 通过；
- 修改后通过；
- 送仲裁。

### 8.4 L3 仲裁机制

触发条件：

- L1 自评低置信度；
- L2 标记争议；
- 抽查发现重大分歧；
- 分级差异达到预设阈值。

仲裁结果为最终结论，并决定是否纳入 Gold Set。

### 8.5 一致性评估指标

至少周期性统计以下指标：

| 指标 | 任务 | 建议目标 |
|---|---|---|
| Cohen's Kappa | 牙位检测 | ≥ 0.85 |
| Dice | 分割 | ≥ 0.80 |
| Weighted Kappa | 分级 | ≥ 0.75 |

若低于目标，应启动复训、抽检和样本返修。

### 8.6 Gold Set 构建要求

Gold Set 用于：

- 新标注员考核；
- 模型最终评估；
- 关键病例答辩展示；
- 错误分析与回归测试。

建议至少包含：

- 多种模态；
- 不同年龄层；
- 不同难度；
- 典型误判场景。

## 九、数据集划分与版本管理

### 9.1 划分原则

- 严格按 `patient_uuid` 划分；
- 不允许同一患者影像跨训练、验证、测试集合；
- 尽量保持年龄、模态、来源、类别分布的基本平衡。

### 9.2 推荐划分比例

| 集合 | 比例 | 用途 |
|---|---|---|
| 训练集 | 70% | 模型训练 |
| 验证集 | 15% | 调参与早停 |
| 测试集 | 15% | 最终评估 |
| 外部验证集 | 独立 | 泛化能力评估 |

### 9.3 版本管理要求

每次数据变更必须发布新版本，采用语义化版本号：

- 主版本：结构重大变化；
- 次版本：新增样本；
- 修订号：修复错误或补充元数据。

每个版本需配套：

- `dataset_metadata.json`
- `dataset_card.md`
- `dataset_changelog.md`
- 样本统计表
- 校验和文件

### 9.4 Dataset Card 建议内容

- 数据来源与授权情况；
- 模态分布；
- 类别分布；
- 标注流程；
- 适用范围与限制；
- 潜在偏差；
- 已知问题；
- 推荐使用方式。

## 十、数据回流与模型训练准入

### 10.1 医生修正数据不直接进入训练

修正数据必须经过以下步骤后，才可进入训练：

1. 业务系统记录修正；
2. 脱敏导出；
3. 数据管理员检查字段完整性；
4. 临床顾问确认样本有效性；
5. 合并至候选训练池；
6. 固化训练版本号。

### 10.2 准入条件

满足以下条件的回流样本方可纳入训练：

- 已完成脱敏；
- 标注字段完整；
- 至少经过一次临床复核；
- 来源清晰且可追溯；
- 不存在合规疑点。

### 10.3 训练后治理要求

- 记录训练数据版本号；
- 记录模型输出指标与错误分析；
- 候选模型须经过离线评估与人工审批后方可上线；
- 保留旧版本回滚能力。

## 十一、证据链与答辩要求

### 11.1 数据侧必须准备的材料

- 伦理批件或授权证明；
- 知情同意模板；
- 脱敏流程说明；
- 数据来源清单；
- 样本统计表；
- 标注一致性报告；
- 数据版本清单；
- Dataset Card。

### 11.2 答辩口径要求

必须区分以下三类表述：

- **已完成**：已经真实获取、处理、统计、验证；
- **在推进**：已经有合作或样本但尚未完全成型；
- **规划目标**：尚未达成，仅为后续目标。

不得把目标规模说成已完成规模，不得把部分样本说成多中心大样本。

## 十二、角色与职责

| 角色 | 职责 |
|---|---|
| 项目负责人 | 整体推进、伦理协调、资源统筹 |
| 数据管理员 | 脱敏、版本管理、抽检、导出审批 |
| 采集协调员 | 现场采集、知情同意、质量初筛 |
| L1 标注员 | 初级标注 |
| L2 医生 | 复核与纠错 |
| L3 专家 | 仲裁与 Gold Set 把关 |

## 十三、应急与变更管理

### 13.1 撤回申请处理

受试者提出撤回时，应根据协议和实际条件，及时完成：

- 业务运行层数据定位；
- 可删除数据删除；
- 训练计划中样本剔除登记；
- 处理结果留档。

### 13.2 数据泄露应急

一旦怀疑发生泄露，应立即：

1. 暂停相关导出与外部访问；
2. 定位受影响数据范围；
3. 通知项目负责人和合作单位；
4. 形成事件记录与整改报告。

### 13.3 规范变更流程

本规范任何修改均需：

- 记录修订原因；
- 更新版本号；
- 留存旧版本；
- 经项目负责人、数据管理员、临床顾问确认。
