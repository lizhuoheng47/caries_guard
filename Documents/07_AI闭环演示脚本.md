# AI 闭环演示脚本

## 1. 演示目标

用 5 到 8 分钟展示 CariesGuard 的核心 AI 闭环，而不是普通后台流程。

主线固定为：

**影像分析 -> uncertainty 复核 -> RAG 解释 -> 风险建议 -> 医生反馈**

## 2. 演示原则

1. 先讲 AI 问题，再讲业务承接
2. 先讲证据链，再讲最终结果
3. 先讲可信边界，再讲功能数量
4. 不从登录页和列表页开始

## 3. 演示前准备

- Java Backend、Python AI/RAG、MySQL、Redis、RabbitMQ、MinIO、向量检索服务全部可用
- 准备四组标准样例：
  - 低风险正常样例
  - 中度龋损样例
  - 高 uncertainty 样例
  - 医生修正样例

## 4. 标准演示路径

### 第一步：场景导入

直接进入一个病例或分析任务详情页。

讲解词：

“我们的作品面向龋病筛查场景，核心不是后台管理，而是围绕影像分析、知识解释和医生复核构建医疗 AI 辅助决策闭环。”

### 第二步：展示影像分析结果

重点展示：

- 原始影像
- 分割 / 热力图 overlay
- `gradingLabel`
- `confidenceScore`
- `uncertaintyScore`
- `rawResultJson`

### 第三步：展示 uncertainty 触发复核

重点展示：

- `needsReview`
- `reviewReason`
- `doctorReviewRequiredReason`

讲解词：

“系统不会把所有 AI 输出都直接当成可靠结论。高 uncertainty 或高风险样本会自动进入复核流程。”

### 第四步：展示 RAG 解释

建议问题：

- 为什么这个病例建议重点复查？
- C2 分级通常意味着什么？
- 高 uncertainty 样本应该如何处理？

重点展示：

- `answer`
- `citations`
- `retrievedChunks`
- `safetyFlags`
- `caseContextSummary`

### 第五步：展示风险评估

重点展示：

- `riskLevel`
- `riskFactors`
- `followUpRecommendation`
- `evidenceRefs`

### 第六步：展示医生修正反馈

重点展示：

- AI 原分级
- 医生修正分级
- 修正理由
- 是否纳入评估池

### 第七步：展示指标看板

重点展示：

- 今日分析任务数
- 平均推理耗时
- 高 uncertainty 占比
- 复核通过率
- RAG 调用量
- 知识命中率
- 医生采纳率

## 5. 必须避免的错误

- 从登录页开始演示
- 只展示最终标签，不展示证据链
- 把 RAG 演示成普通聊天机器人
- 忽略 uncertainty
- 弱化医生最终决策边界

## 6. 5 分钟精简版

### 0:00 - 0:30

场景导入

### 0:30 - 1:30

影像分析结果 + visual assets

### 1:30 - 2:15

高 uncertainty 样例 + 自动复核

### 2:15 - 3:15

RAG 问答 + citations

### 3:15 - 4:00

风险评估 + 复查建议

### 4:00 - 4:40

医生修正反馈

### 4:40 - 5:00

总结：

“我们的核心不是单点模型，而是从分析、解释、复核到反馈的可追踪 AI 闭环。”
