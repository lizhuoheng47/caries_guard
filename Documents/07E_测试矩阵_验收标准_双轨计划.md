# 07E 测试矩阵、验收标准、双轨计划

## 1. 测试矩阵

| 类型 | 目标 | 责任方 | 最小样例数 | 通过标准 | 输出物 |
| --- | --- | --- | --- | --- | --- |
| 单元测试 | DTO、服务、融合、拒答规则稳定 | Java / Python | 每个核心服务至少 1 组 | 无断言失败 | `pytest` / `mvn test` 报告 |
| 集成测试 | Java -> Python 契约与 OpenSearch / Neo4j / MySQL 写入 | 后端 | 至少 5 组主链路 | 状态、索引、图谱一致 | 集成测试报告 |
| E2E | 前端 -> Java -> Python 主流程闭环 | 全栈 | 至少 3 组 | 上传、审核、发布、问答可用 | E2E 清单 |
| 安全测试 | 越权问答、prompt injection、隐私输入 | Python / Java | 至少 6 组 | 必须拒答或保守输出 | 安全回归清单 |
| 检索效果测试 | citation 命中率、graph path 命中率 | Python | 至少 20 个问题 | 指标达到阈值 | `rag_eval_run.metric_json` |
| 拒答与保守表达测试 | 无证据、危险问题、诊断请求 | Python | 至少 10 组 | `refusalReason` 与 `safetyFlags` 正确 | 拒答样例集 |
| 发布回滚测试 | 发布唯一性、回滚显式留痕 | Java / Python | 至少 3 组 | 仅 1 个 published 版本 | 回滚记录核验 |
| 数据一致性测试 | MySQL / OpenSearch / Neo4j 对齐 | 后端 | 至少 5 个文档版本 | chunk 数、实体数、关系数可追溯 | 差异报告 |

## 2. 评估集冻结

- `rag_eval_dataset`

- `expected_risk_level`

结果固定字段：

- `citation_hit_flag`
- `graph_hit_flag`
- `refusal_hit_flag`
- `hallucination_flag`
- `latency_ms`
- `metric_json`

## 3. 指标阈值

默认验收阈值：

1. `citationAccuracy >= 0.70`
2. `citationCoverage >= 0.60`
3. `graphPathHitRate >= 0.50`
4. `refusalPrecision >= 0.90`
5. `groundednessRate >= 0.85`
6. `avgLatencyMs <= 3000`

## 4. 比赛交付目标

1. 前端可演示知识导入、审核、发布、问答、日志查看、评估运行。
2. Java 外部接口只保留 `/api/v1/kb/**` 与 `/api/v1/rag/**`。
3. Python 支持 OpenSearch、Neo4j、RAG 三路检索真实链路。
4. 所有回答带引用、置信度、安全标记、追踪号。

## 5. 完整工程交付目标

1. 完成多角色权限矩阵与更细粒度审计。
2. 增强图谱实体类型与规则库。
3. 完成更多评估集与自动回归流水线。
4. 增强失败告警、死信队列与后台任务运维能力。
