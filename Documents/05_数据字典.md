# 数据字典

## 1. 数据总体设计

系统采用双库设计：

| 数据库 | 负责人 | 职责 |
| --- | --- | --- |
| `caries_biz` | Java | 业务主数据、状态机、附件、报告、随访、权限 |
| `caries_ai` | Python | 推理运行日志、知识库、RAG 日志、模型治理、训练数据 |

约束：

- 不跨库建外键。
- 不跨库做迁移。
- 通过业务标识弱关联，如 `task_no`、`case_id`、`case_no`、`image_id`、`attachment_id`。

## 2. 业务对象关系

主关系如下：

```text
pat_patient
  -> med_visit
    -> med_case
      -> med_image_file
      -> ana_task_record
        -> ana_result_summary
        -> ana_visual_asset
        -> med_risk_assessment_record
      -> rpt_record
      -> fup_plan -> fup_task -> fup_record
```

AI 侧关系如下：

```text
ai_infer_job
  -> ai_infer_job_image
  -> ai_infer_artifact
  -> ai_callback_log

kb_knowledge_base
  -> kb_document
    -> kb_document_chunk

rag_session
  -> rag_request_log
    -> rag_retrieval_log
    -> llm_call_log

mdl_model_version
  -> mdl_model_eval_record
  -> mdl_model_approval_record

trn_dataset_snapshot
  -> trn_dataset_sample
  -> ann_annotation_record
```

## 6. 关键枚举与业务语义

### 6.1 Java 业务状态

- 病例状态示例：`CREATED`, `QC_PENDING`, `ANALYZING`, `REVIEW_PENDING`, `REPORT_READY`, `FOLLOWUP_REQUIRED`, `CLOSED`
- 分析任务状态示例：`PENDING`, `RUNNING`, `SUCCESS`, `FAILED`
- 报告状态示例：草稿、已生成、已签发、已导出
- 随访状态示例：计划中、待执行、已完成、已关闭

### 6.2 AI 结果语义

- 分级标签：`C0`, `C1`, `C2`, `C3`
- 实现类型：`MOCK`, `HEURISTIC`, `ML_MODEL`, `RAG`, `GENERAL_LLM`
- 运行模式：`mock`, `hybrid`, `real`

### 6.3 通用审计字段

绝大多数表包含以下通用字段：

- `org_id`
- `status`
- `deleted_flag`
- `remark`
- `created_by`
- `created_at`
- `updated_by`
- `updated_at`

## 7. 数据治理约束

- `caries_ai` 不保存患者明文身份。
- 敏感信息在业务库中采用密文、哈希和脱敏展示并存。
- AI 详细结果优先写入 JSON 留痕字段。
- 需要稳定查询、统计或状态推进时，再固化为实体列。
- 导出和中间可视化文件必须设置生命周期策略。
