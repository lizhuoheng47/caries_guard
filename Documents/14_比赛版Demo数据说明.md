# 14. 比赛版 Demo 演示指导手册

本文档是针对 CariesGuard "竞赛版/评委版" 的正式演示标准指导。通过执行 `competition-up` 启动命令，系统会放弃耗时且易挥发的真实 E2E 模型推理链路，直接落位为**完全确定性的 Demo Baseline**，从而实现**系统的「零人工」即刻演示就绪状态**。

## 1. 原则与演示边界

1. **绝对确定性**：评委在体验端看到的所有演示样例（包含机构、登录账号、分诊流转、诊断结论及核心 RAG 释义）均受硬编码种子（Seed Data）强保护，可完美复现与多次重演。
2. **免编排就绪**：一切依赖项均在 Docker Compose `up` 生命周期内通过后置 Hook 自动拉起与挂载，演示者无需调用额外脚本手动造库造片。

## 2. 冻结的演示身份字典

为了避免全层级间的溯源断链，系统锁死了以下核心数据的唯一标识：

| 实体分类 | 锁定主键 | 业务表现标识 |
| :--- | :--- | :--- |
| **机构配置** | `id=100001` | 编码：`ORG-001`，名称：`Demo Clinic` |
| **标准接诊医生** | `id=1001` | 账号：`demo_doctor_01` (Dr. Smith) |
| **质控复核医生** | `id=1002` | 账号：`demo_doctor_02` (Dr. Lee) |
| **患者画像 A** | `id=2001` | 患者号：`P-2026-001` (男性/常规筛查) |
| **患者画像 B** | `id=2002` | 患者号：`P-2026-002` (女性/复杂病史) |
| **低风险病例范例** | `id=3001` | 病历号：`CA-2026-LOW` |
| **高风险/高不确定性范例** | `id=3002` | 病历号：`CA-2026-HIGH` |
| **知识库域约束** | N/A | `kb_code='COMPETITION_DEMO_KB'` |

> 👉 **注**: 所有密码已强制同步为 `123456`，通过标准 BCrypt 解析。涉及高敏信息部分采用固化密文或 `NULL` 兜底保证读取不崩。

## 3. 标准演示沙盘流程

在启动时（通过 `competition-up`），系统自动使用 SQL 与 Python 脚本预置了两套核心样例流：

### 视角 A：自动落锤闭环（低不确定度）
- **触发途径**：以 `demo_doctor_01` 登录，打开病历 `CA-2026-LOW`。
- **AI 既定事实**：系统强制提取 `{"overall_highest_severity": "LOW_RISK"}` 并携带完整边框数据。由于不确定度较低，报告自动盖戳。
- **演示话术**："各位评委请看此类浅龋病历，模型置信度极高，系统因此未做任何阻拦，快速释放为 `REPORT_READY` 状态，节约了临床时间。"

### 视角 B：阈值红绿灯拦截（高不确定度）
- **触发途径**：注销并以高级权限医师 `demo_doctor_02` 登录，定位到 `CA-2026-HIGH` 待办。
- **AI 既定事实**：在 AI 特征结果 `raw_result_json` 中，系统锁定了 `HIGH_RISK` 并将预设警报位打开。
- **演示话术**："但在面对边缘轮廓不清的病例，由于置信度跌破阀值被评为极高风险，系统主动触发阻断，将其推举为 `REVIEW_PENDING` 需高年资医生签批，从而守住医疗底线。"
- **Review 差异与反馈范例**：在此用例下，AI 原本标记为 C3 但带有高不确定性，医生复核后选择同意分级，但修正了随访建议。演示闭环最终以人工决议为准并落入反馈表。

## 4. 问答(RAG)解释引擎：定点狙击演示包

为了体现知识管理组件对 AI “黑盒”的解构能力，Python 核心在初始化时已利用 `seed_demo_knowledge.py` 埋入了四类脱敏知识：
1. **《2026版国家口腔健康指南》**（指南来源）
2. **预防龋齿患者教育手册**（科普素材）
3. **高糖饮食与低氟环境风险因子**（业务定义库）
4. **内部规则：系统高不确定性复核机制说明**（内部策略库）

在 AI 结果或 RAG 界面中逐字粘贴以下问题，系统可确保精准召回知识块，呈现高医学严谨度的保守解释（均挂载确定的知识追溯）：

- **👉 演示 Prompt 1（触及分栏规则）**
  - **角色**: 医生
  - **问题**: "为什么系统将病灶部位判定为中度龋齿？"
  - **预期引用**: `DEMO-DOC-001` (国家指南)

- **👉 演示 Prompt 2（系统级风险控制解释）**
  - **角色**: 管理者/质控医生
  - **问题**: "病例 CA-2026-HIGH 为什么分诊报警要求强制复核？"
  - **预期引用**: `DEMO-DOC-005` (内部规则)

- **👉 演示 Prompt 3（患者关怀解释）**
  - **角色**: 患者 / 接诊医生替患者问
  - **问题**: "我被诊断为中度龋齿，平时需要注意什么？"
  - **预期引用**: `DEMO-DOC-002` (患者教育手册)

- **👉 演示 Prompt 4（风险因子解读）**
  - **角色**: 医生
  - **问题**: "高糖饮食患者的风险评分如何判定？"
  - **预期引用**: `DEMO-DOC-003` (风险因子定义)

## 5. 收尾与重置动作

在答辩或预演完毕后，仅需在根目录执行 `./scripts/competition-reset.sh`（或 `.ps1`）。重置脚本将重建数据字典与 MinIO 占位图，完全抹除人工操作带来的“脏数据”，为裁判重新体验铺设纯净底座。
