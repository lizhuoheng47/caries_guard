# 14. 比赛版 Demo 演示指导手册

本文档是针对 CariesGuard "竞赛版/评委版" 的正式演示标准指导。通过执行 `competition-up` 启动命令，系统会放弃耗时且易挥发的真实 E2E 模型推理链路，直接落位为**完全确定性的 Demo Baseline**，从而实现**系统的「零人工」即刻演示就绪状态**。

## 1. 原则与演示边界

1. **绝对确定性**：评委在体验端看到的所有演示样例（包含机构、登录账号、分诊流转、诊断结论及核心 RAG 释义）均受硬编码种子（Seed Data）强保护，可完美复现与多次重演。
2. **免编排就绪**：一切依赖项均在 Docker Compose `up` 生命周期内通过后置 Hook 自动拉起与挂载，演示者无需调用额外脚本手动造库造片。

## 2. 冻结的演示身份字典

为了避免全层级间的溯源断链，系统锁死了以下核心数据的唯一标识：

| 实体分类 | 锁定主键 | 业务表现标识 |
| :--- | :--- | :--- |
| **机构配置** | `id=100` | 编码：`ORG-001`，名称：`Demo Clinic` |

## 3. 标准演示沙盘流程

在启动时，我们预置了两类截然不同的诊断链路，用于比对自动闭环效率与人机协同的边界拦截：

### 视角 A：自动落锤闭环（低不确定度）
- **触发途径**：以 `demo_doctor_01` 登录，打开病历 `CA-2026-LOW`。
- **AI 既定事实**：系统强制提取 `{"overall_highest_severity": "LOW_RISK"}` 并携带完整边框数据。由于不确定度较低，报告自动盖戳。
- **演示话术**："各位评委请看此类浅龋病历，模型置信度极高，系统因此未做任何阻拦，快速释放为 `REPORT_READY` 状态，节约了临床时间。"

### 视角 B：阈值红绿灯拦截（高不确定度）
- **触发途径**：注销并以高级权限医师 `demo_doctor_02` 登录，定位到 `CA-2026-HIGH` 待办。
- **AI 既定事实**：在 AI 特征结果 `raw_result_json` 中，系统锁定了 `HIGH_RISK` 并将预设警报位打开。
- **演示话术**："但在面对边缘轮廓不清的病例，由于置信度跌破阀值被评为极高风险，系统主动触发阻断，将其推举为 `REVIEW_PENDING` 需高年资医生签批，从而守住医疗底线。"

## 4. 问答(RAG)解释引擎：定点狙击演示包

为了体现知识管理组件对 AI “黑盒”的解构能力，Python 核心预埋了最极端的确定性 RAG 响应流（对应 `DOC-001` 至 `DOC-005`）。

**演示要求**：在智能提问框中逐字粘贴以下固定 Prompt 提问，以获得最专业的对应导向：

- **👉 演示 Prompt 1（触及分栏规则）**
  - **Q输入**: "为什么系统将病灶部位判定为中度龋齿？"
  - **预期 A (DOC-001)**: "根据《2026版国家口腔健康指南》，当透射区越过牙本质浅层进入深层，但在影像学上尚保留完整轮廓的，判定为中度龋齿（Medium Caries），必须由执业医师最终确认。"

- **👉 演示 Prompt 2（触及前端拦截解释器）**
  - **Q输入**: "病例 CA-2026-HIGH 为什么分诊报警要求强制复核？"
  - **预期 A (DOC-005)**: "当不确定性概率（Uncertainty Score）> 0.35 时，由于其处于模型可信度的边缘地带，系统将拒绝出具最终报告，强行引发状态升级（REPORT_READY -> REVIEW_PENDING），自动流转到复核队列要求高级资质人工介入。"

- **👉 演示 Prompt 3（触及随访控制法则）**
  - **Q输入**: "边缘邻面龋被判定后，患者应该什么时候转为复查？"
  - **预期 A (DOC-004)**: "高复查指数的边缘邻面龋必须在15天内进行首轮强制干预和复核确认（15-day Recall Policy），若逾期将自动升级预警级别直到主治医生响应。"

## 5. 收尾与重置动作

在答辩或预演完毕后，仅需在根目录执行 `./scripts/competition-reset.sh`（或 `.ps1`）。重置脚本将严密释放 Docker 分卷并连根拔起生成的所有 Dummy Assets 图床缓存，为下一轮裁判评审铺垫完全无痕的起始点。
