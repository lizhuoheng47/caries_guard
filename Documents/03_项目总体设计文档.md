# 项目总体设计文档

更新日期：2026-04-15

本文档面向 Java 与 Python 协作开发，描述当前 `backend-java` 已实现的模块边界、核心类、方法、接口与数据库落点。

## 1. 设计目标

当前项目目标是提供龋齿筛查与诊疗辅助业务平台后端，重点是完整业务闭环，而不是单纯模型推理接口。

已实现闭环：

1. 系统登录、权限、角色、菜单、数据权限规则。
2. 患者、就诊、病例、诊断、牙位记录。
3. 影像附件上传、MinIO 对象存储、访问 URL、影像质检。
4. AI 分析任务创建、RabbitMQ 投递、Python 回调、结果入库、幂等保护。
5. 医生修正反馈、训练候选治理字段。
6. 报告模板、报告生成、PDF 附件、导出审计、下载 URL。
7. 随访计划、任务、记录。
8. 业务看板和 AI 模型运行质量看板。

## 3. 模块设计

### 3.1 `caries-system`

职责：认证、当前用户、用户管理、角色管理、菜单管理、字典、配置、数据权限规则。

Controller：

| 类名 | API 前缀 | 方法 |
| --- | --- | --- |
| `AuthController` | `/api/v1/auth` | `login`、`currentUser`、`currentPermissions` |
| `SystemAdminController` | `/api/v1/system` | `pageUsers`、`getUser`、`createUser`、`updateUser`、`listRoles`、`getRole`、`createRole`、`updateRole`、`listMenus`、`getMenu`、`createMenu`、`updateMenu`、`listDataPermissionRules`、`createDataPermissionRule`、`updateDataPermissionRule` |
| `SystemMetadataController` | `/api/v1/system` | `listDictTypes`、`listDictItems`、`getConfig` |
| `SystemSupportController` | `/api/v1/system` | `ping` |

AppService：

| 类名 | 职责 |
| --- | --- |
| `AuthAppService` | 登录、当前用户、权限列表 |
| `SystemUserCommandAppService` | 用户新增/修改 |
| `SystemRoleCommandAppService` | 角色新增/修改 |
| `SystemMenuCommandAppService` | 菜单新增/修改 |
| `SystemDataPermissionRuleAppService` | 数据权限规则管理 |
| `SystemQueryAppService` | 字典、配置查询 |
| `SystemAdminQueryAppService` | 后台管理查询 |

数据库基线已补业务角色、菜单、角色菜单关系和数据权限规则种子。

Controller：

| 类名 | API 前缀 | 方法 |
| --- | --- | --- |

### 3.3 `caries-image`

职责：附件上传、对象存储、访问 URL、影像记录、质检记录。

Controller：

| 类名 | API 前缀 | 方法 |
| --- | --- | --- |
| `FileController` | `/api/v1/files` | `upload`、`accessUrl`、`content` |
| `CaseImageController` | `/api/v1/cases/{caseId}/images` | `create`、`list`、`detail` |
| `ImageQualityCheckController` | `/api/v1/images/{imageId}/quality-checks` | `save`、`getCurrent` |

对象存储：

| 抽象/实现 | 说明 |
| --- | --- |
| `ObjectStorageService` | 统一对象存储接口，方法为 `store`、`load`、`delete` |
| `MinioObjectStorageClient` | `caries.storage.provider=MINIO`，当前默认实现，负责受配置控制的 bucket 检查、put/get/stat/delete |
| `ImageObjectStorageServiceAdapter` | `provider-code=LOCAL_FS`，本地文件系统兼容实现 |
| `StorageProperties` | 读取 `caries.storage` 配置，包括 provider、endpoint、四个 bucket、预签名过期时间和自动建桶开关 |

`AttachmentAppService` 方法：

| 方法 | 说明 |
| --- | --- |
| `upload(MultipartFile)` | 计算 MD5、去重、写对象存储、写 `med_attachment` |
| `createAccessUrl(Long, HttpServletRequest)` | 给前端生成 MinIO presigned URL |
| `createAccessUrl(Long, String)` | 兼容签名，当前生成 MinIO presigned URL |
| `createInternalAccessUrl(Long)` | 给内部 AI 服务生成 MinIO presigned URL |
| `resolveLocalStoragePath(String, String)` | LOCAL_FS 场景解析本地路径 |
| `loadContent(Long, Long, String)` | 受控代理兜底入口，校验签名和过期时间后读取对象内容 |

### 3.4 `caries-analysis`

职责：分析任务、AI 事件投递、回调处理、结果摘要、风险评估、视觉资产、修正反馈。

Controller：

| 类名 | API 前缀 | 方法 |
| --- | --- | --- |
| `AnalysisTaskController` | `/api/v1/analysis/tasks` | `createAnalysisTask`、`retryAnalysisTask`、`getAnalysisTaskDetail`、`pageAnalysisTasks` |
| `CaseAnalysisAliasController` | `/api/v1/cases/{caseId}` | `createAnalysisFromCase`、`submitCorrectionFromCase` |
| `AnalysisCallbackController` | `/api/v1/internal/ai/callbacks` | `receiveAnalysisResultCallback` |
| `CorrectionFeedbackController` | `/api/v1/analysis/corrections` | `submitCorrectionFeedback` |

关键 DTO/Command：

| 类名 | 当前用途 |
| --- | --- |
| `CreateAnalysisTaskCommand` | 前端/业务侧创建分析任务 |
| `RetryAnalysisTaskCommand` | 重试分析任务 |
| `AiAnalysisRequestDTO` | Java 发送给 Python 的任务载荷 |
| `AiAnalysisResultCallbackCommand` | Python 回调 Java 的冻结载荷 |
| `AiVisualAssetDTO` | 回调中的视觉资产结构 |
| `RiskAssessmentDTO` | 回调中的风险评估结构 |
| `SubmitCorrectionFeedbackCommand` | 医生修正反馈提交 |

当前回调契约名称固定为 `AiAnalysisResultCallbackCommand`。

`AiAnalysisRequestDTO.ImageItem` 现在包含：`imageId`、`attachmentId`、`imageTypeCode`、`bucketName`、`objectKey`、`storageProviderCode`、`attachmentMd5`、`accessUrl`、`accessExpireAt`、`localStoragePath`。

`AiAnalysisResultCallbackCommand` 现在包含：`taskNo`、`taskStatusCode`、`startedAt`、`completedAt`、`modelVersion`、`summary`、`rawResultJson`、`visualAssets`、`riskAssessment`、`errorMessage`、`traceId`、`inferenceMillis`、`uncertaintyScore`。

关键表：`ana_task_record`、`ana_result_summary`、`ana_visual_asset`、`med_risk_assessment_record`、`ana_correction_feedback`、`ana_model_version_registry`。
