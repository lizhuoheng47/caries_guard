# 05 开发规范

## 1. 命名原则

- 业务实体使用清晰英文；
- 数据库字段使用 `snake_case`；
- Java 类使用 `PascalCase`；
- Java 方法和变量使用 `camelCase`；
- JSON 对外字段使用 `camelCase`；
- AI 原始输出统一放入 `rawResultJson`。

## 2. Java 分层约定

| 层 | 职责 |
| --- | --- |
| Controller | HTTP 入参、鉴权、响应 |
| AppService | 用例编排 |
| DomainService | 业务规则 |
| Repository / Mapper | 数据访问 |
| IntegrationClient | 外部服务调用（Python / RAG / MinIO / MQ） |

Java 模块按业务域划分：

- `caries-system`：用户、角色、组织、权限；
- `caries-patient`：患者、就诊、病例；
- `caries-image`：影像和附件；
- `caries-analysis`：AI analysis task、callback、summary、visual asset；
- `caries-report`：报告；
- `caries-followup`：随访；
- `caries-integration`：外部服务集成；
- `caries-boot`：启动入口。

### 建议关键类

Analysis：`AnalysisTaskController` / `AnalysisTaskAppService` / `AnalysisCallbackController` / `AnalysisCallbackAppService` / `VisualAssetService` / `ReviewRoutingService`。

RAG：`RagController` / `RagAppService` / `PythonRagClient` / `RagResponseAssembler`。

## 3. Java 禁止逻辑

Java 不得：

- 直接实现影像模型；
- 直接调用通用大模型 provider；
- 管理微调训练任务；
- 修改 `caries_ai`；
- 将 RAG 文本直接写成临床最终结论；
- 在日志输出敏感明文；
- 破坏 callback 兼容性。

## 4. Callback 兼容约定

新增字段优先进入：

- `rawResultJson`
- `assessment_report_json`
- RAG response 扩展字段

避免修改：

- callback 顶层 DTO；
- `visualAssets` 顶层结构；
- Java 业务表主键和状态字段。

## 5. 日志约定

必须记录：`taskNo` / `traceId` / runtime mode / `implType` / `errorCode` / `inferenceMillis`。

不得记录：明文身份证、明文手机号、不必要的患者姓名、大段 prompt 敏感内容。

## 6. RAG 字段命名

建议字段：

- `knowledgeBaseCode`
- `documentCode`
- `chunkId`
- `retrievalScore`
- `citations`
- `safetyFlags`
- `llmProviderCode`
- `llmModelName`

RAG 回答必须保留引用，不能只返回纯文本。

## 7. Phase 5C 字段

统一字段（大小写不得漂移）：

- `gradingMode`
- `gradingImplType`
- `gradingLabel`
- `confidenceScore`
- `uncertaintyMode`
- `uncertaintyImplType`
- `uncertaintyScore`
- `needsReview`

## 8. 数据集与标注规范

### 8.1 数据类型

- **影像评估数据**：用于 quality / detection / segmentation / grading / uncertainty 校准。必须标注影像类型、牙位、病灶区域、分级标签 `C0/C1/C2/C3`、标注医生、复核状态、标注版本、图像质量问题。
- **知识图库数据**：每条知识必须记录来源、版本、审核人、适用范围和失效时间。
- **业务反馈数据**：AI 分级与医生结论差异、high uncertainty 是否正确触发复核、报告修改记录、随访结果、RAG 回答评分、医生采纳率。

### 8.2 Uncertainty 标注维度

标注者一致性、图像质量问题、病灶边界不清、牙位遮挡、分级边界样本、是否需要复核。

### 8.3 合规要求

- 不采集明文身份信息进入 AI 数据集；
- 影像和病例必须脱敏；
- RAG 知识来源必须可授权；
- 评估集变更必须留版本；
- 医生标注 / 复核操作可追溯；
- `caries_ai` 不保存患者明文身份；
- RAG 知识库不得混入未脱敏病例全文。

## 9. 回归要求

修改以下内容必须跑 Docker E2E：

- callback DTO；
- `rawResultJson` 字段；
- MinIO visual asset；
- grading / uncertainty；
- RAG response 结构；
- 数据库 migration。

## 10. AI 协同原则

- Java 管业务，Python 管 AI / RAG；
- 通用大模型只生成解释和问答，不修改业务状态；
- 规则和阈值决定业务状态；
- 医生复核决定最终医疗意见；
- `real` 模式失败必须显式失败；
- 所有 AI 结果必须可追溯。
