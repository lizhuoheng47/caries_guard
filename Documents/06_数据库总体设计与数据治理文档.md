# 06 数据库总体设计与数据治理文档

## 1. 总体原则

CariesGuard 使用双库设计：

- `caries_biz`：Java 业务主库；
- `caries_ai`：Python AI/RAG 运行库。

两库职责清晰，不跨库迁移，不跨库写主数据。

## 2. 数据治理目标

- 业务数据权威明确；
- AI 输出可追溯；
- RAG 回答可引用；
- 隐私数据最小化；
- 模型/算法/知识版本可审计；
- 失败和回退路径可复盘。

## 3. Java 业务库

主要域：

- 用户和组织；
- 患者和就诊；
- 病例和影像；
- 附件；
- analysis task；
- summary 和 visual asset；
- report；
- review；
- followup。

## 4. Python AI/RAG 库

主要域：

- AI 推理任务；
- 图像级结果；
- callback 日志；
- RAG 知识库；
- 文档 chunk；
- 检索日志；
- 通用大模型调用日志；
- 模型/算法/知识版本治理。

## 5. 通用大模型数据治理

不保存通用大模型微调训练数据作为在线主数据。

必须治理：

- 知识来源；
- 知识版本；
- 检索证据；
- prompt 摘要；
- response 摘要；
- 引用准确率；
- 人工评估；
- 安全拒答。

## 6. JSON 留痕

AI 结果优先保留完整 JSON：

- Java：`ana_result_summary.raw_result_json`
- Python：`ai_infer_job.result_json`
- Python：`ai_infer_job_image.result_json`
- RAG：`rag_request_log`、`rag_retrieval_log`、`llm_call_log`

## 7. 隐私要求

- `caries_ai` 不保存患者明文身份；
- prompt 不包含非必要敏感信息；
- 日志只保存必要摘要；
- MinIO bucket 私有；
- 导出数据有生命周期。

## 8. 数据质量

关键指标：

- task 成功率；
- callback 成功率；
- visual asset 完整率；
- grading/uncertainty 留痕完整率；
- RAG 检索命中率；
- 大模型回答引用准确率；
- 医生复核一致率。
