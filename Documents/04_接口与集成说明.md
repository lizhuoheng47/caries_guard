# 接口与集成说明

> 2026-04-19 起，知识库与 RAG 的工程冻结规范以 `07_知识图库与AI全量实施方案.md` 及其附录为准；本文保留为兼容概览。

## 1. 集成原则

- Java 是业务入口和状态权威。
- Python 是 AI/RAG 能力提供方。
- Analysis 请求通过 RabbitMQ 投递。
- Analysis 结果通过 HTTP callback 回传。
- RAG、知识库、健康检查通过 HTTP 调用。
- 新增 AI 细节优先进入 JSON 字段，避免频繁破坏稳定 DTO。

## 4. MQ 集成

默认配置来自 `docker-compose.yml`：

- Exchange：`caries.analysis.exchange`
- Queue：`caries.analysis.requested.queue`
- Routing Key：`analysis.requested`

流程：

1. Java 创建任务并投递 MQ。
2. Python Worker 消费任务。
3. Python 处理完成后调用 Java callback。

## 5. Analysis 请求契约

Java 向 Python 投递的分析请求应至少包含：

- `taskNo`
- `traceId`
- `caseId` / `caseNo`
- `patientId`
- `orgId`
- `modelVersion`
- `images`
- `callbackUrl`

设计要求：

- 影像访问应通过 MinIO bucket/object 或已签名访问地址。
- 不应在请求中携带不必要的敏感明文字段。
- `taskNo` 和 `traceId` 必须可以串起完整链路。

## 6. Analysis Callback 契约

Python 回传 Java 的 callback 关键字段：

- `taskNo`
- `taskStatusCode`
- `startedAt`
- `completedAt`
- `modelVersion`
- `summary`
- `rawResultJson`
- `visualAssets`
- `riskAssessment`
- `uncertaintyScore`
- `inferenceMillis`
- `errorCode`
- `errorMessage`
- `traceId`

Java 侧处理要求：

- 校验签名
- 校验任务存在性
- 幂等处理
- 失败状态真实落库
- 兼容 `rawResultJson` 的新增字段

## 7. `rawResultJson` 关键字段

当前实现中建议稳定保留：

- `pipelineVersion`
- `mode`
- `qualityMode`
- `qualityImplType`
- `toothDetectionMode`
- `toothDetectionImplType`
- `segmentationMode`
- `segmentationImplType`
- `gradingMode`
- `gradingImplType`
- `gradingLabel`
- `confidenceScore`
- `uncertaintyMode`
- `uncertaintyImplType`
- `uncertaintyScore`
- `needsReview`
- `visualAssets`

其中 Java 业务状态主要依赖：

- `gradingLabel`
- `uncertaintyScore`
- `needsReview`

## 8. RAG 输出契约

Python RAG 返回的核心字段：

- `answer`
- `citations`
- `retrievedChunks`
- `knowledgeVersion`
- `confidence`
- `safetyFlags`
- `refusalReason`
- `traceId`

约束：

- 必须支持引用回显。
- 不应把回答包装成最终诊断结论。
- 当证据不足时应允许保守回答或拒答。
- 前端只调用 Java 暴露的 `/api/v1/kb/**` 与 `/api/v1/rag/**`，不允许直连 Python。

## 9. 存储与附件规则

- Java 是附件主数据权威。
- Python 负责生成 visual asset 文件和元数据。
- Java 将 visual asset 统一登记到 `med_attachment` 和 `ana_visual_asset`。
- 存储定位使用 `bucket_name + object_key`。

## 10. 兼容性要求

- Java callback DTO 需要容忍未知 JSON 字段。
- `visualAssets` 顶层结构应保持稳定。
- Python 不直接写入 `caries_biz` 业务表。
- Java 不直接写入 `caries_ai` 推理与检索日志表。
