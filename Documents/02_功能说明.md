# 功能说明

## 1. 功能总览

CariesGuard 当前可用能力可以分为八个模块：

1. 身份与权限
2. 患者、就诊与病例
3. 影像与附件管理
4. AI 分析任务与结果回流
5. 医生复核与纠偏反馈
6. 报告与 RAG 解释
7. 随访管理
8. 看板与模型治理

## 2. 身份与权限

Java `caries-system` 模块提供：

- 登录、当前用户信息、权限查询
- 用户、角色、菜单、数据权限规则管理
- 字典与配置项查询
- 比赛模式下的菜单和权限收缩

对应接口示例：

- `POST /api/v1/auth/login`
- `GET /api/v1/auth/me`
- `GET /api/v1/auth/permissions`
- `GET /api/v1/system/users`
- `GET /api/v1/system/roles`
- `GET /api/v1/system/dicts`

## 3. 患者、就诊与病例

Java `caries-patient` 模块提供：

- 患者主档维护
- 就诊记录查询
- 病例详情查询
- 诊断记录录入
- 牙位记录录入
- 病例状态流转

对应接口示例：

- `GET /api/v1/patients/{patientId}`
- `PUT /api/v1/patients/{patientId}`
- `GET /api/v1/visits/{visitId}`
- `GET /api/v1/cases/{caseId}`
- `POST /api/v1/cases/{caseId}/diagnoses`
- `POST /api/v1/cases/{caseId}/tooth-records`
- `POST /api/v1/cases/{caseId}/status-transition`

## 4. 影像与附件管理

Java `caries-image` 模块提供：

- 影像文件与附件上传
- 附件访问地址签发
- 影像详情查询
- 当前质检结果查询

对应接口示例：

- `POST /api/v1/files/upload`
- `GET /api/v1/files/{attachmentId}/access-url`
- `GET /api/v1/files/{attachmentId}/content`
- `GET /api/v1/cases/{caseId}/images/detail/{imageId}`
- `GET /api/v1/images/{imageId}/quality-checks/current`

## 5. AI 分析任务与结果回流

Java `caries-analysis` 和 Python 推理服务共同完成：

- 创建分析任务
- 异步投递 MQ
- 下载脱敏影像
- 执行质量检查、检测、分割、分级、不确定性、风险评估
- 生成 visual assets
- 回调 Java 落库
- 查询任务详情与重试失败任务

Java 接口：

- `POST /api/v1/analysis/tasks`
- `GET /api/v1/analysis/tasks`
- `GET /api/v1/analysis/tasks/{taskId}`
- `POST /api/v1/analysis/tasks/retry`
- `POST /api/v1/internal/ai/callbacks/analysis-result`

Python 接口：

- `POST /ai/v1/analyze`
- `POST /ai/v1/quality-check`
- `POST /ai/v1/assess-risk`
- `GET /ai/v1/model-version`
- `GET /ai/v1/health`

关键输出：

- `gradingLabel`
- `uncertaintyScore`
- `needsReview`
- `riskLevel`
- `riskFactors`
- `visualAssets`
- `rawResultJson`

## 6. 医生复核与纠偏反馈

Java 当前支持：

- 病例级复核纠偏入口
- 纠偏反馈记录
- 训练候选样本导出

对应接口示例：

- `POST /api/v1/cases/{caseId}/corrections`
- `POST /api/v1/analysis/corrections/review`
- `POST /api/v1/analysis/corrections/training-candidates/export`

这部分能力用于承接高不确定性结果和医生修正意见，是后续模型治理的主要数据来源。

## 7. 报告与 RAG 解释

Java `caries-report` 与 Python RAG 服务共同提供：

- 医生问答
- 患者解释
- 通用问答入口
- 报告创建、查询、导出
- 知识文档导入与索引重建

Java 接口：

- `POST /api/v1/rag/doctor-qa`
- `POST /api/v1/rag/patient-explanation`
- `POST /api/v1/rag/ask`
- `POST /api/v1/cases/{caseId}/reports`
- `GET /api/v1/cases/{caseId}/reports`
- `GET /api/v1/reports/{reportId}`
- `POST /api/v1/reports/{reportId}/export`

Python 接口：

- `POST /ai/v1/rag/doctor-qa`
- `POST /ai/v1/rag/patient-explanation`
- `POST /ai/v1/rag/ask`
- `POST /ai/v1/knowledge/documents`
- `POST /ai/v1/knowledge/rebuild`

RAG 返回内容包含：

- `answer`
- `citations`
- `retrievedChunks`
- `knowledgeVersion`
- `safetyFlags`

## 8. 随访管理

Java `caries-followup` 模块提供：

- 随访计划创建、取消、关闭
- 随访任务创建、分配、状态更新
- 随访记录登记

对应接口示例：

- `POST /api/v1/cases/{caseId}/followup/plans`
- `GET /api/v1/cases/{caseId}/followup/plans`
- `POST /api/v1/cases/{caseId}/followup/tasks`
- `GET /api/v1/cases/{caseId}/followup/tasks`
- `POST /api/v1/followup/records`

## 9. 看板与模型治理

Java 提供业务看板和运行治理接口：

- 业务统计总览
- 风险分布、病例状态分布、待办统计
- AI 运行时看板
- 模型版本登记、审批、评估
- 数据集快照登记

对应接口示例：

- `GET /api/v1/dashboard/overview`
- `GET /api/v1/dashboard/case-status-distribution`
- `GET /api/v1/dashboard/risk-level-distribution`
- `GET /api/v1/dashboard/model-runtime`
- `POST /api/v1/analysis/model-governance/model-versions`
- `POST /api/v1/analysis/model-governance/model-versions/{modelVersionId}/approval`
- `POST /api/v1/analysis/model-governance/dataset-snapshots`
- `POST /api/v1/analysis/model-governance/evaluations`

## 10. 当前实现边界

- 当前仓库没有独立前端工程，演示主要依赖接口、脚本和看板接口。
- `mock` 模式是默认、最稳定的演示方式。
- 影像分析与 RAG 主链已具备较完整闭环。
- 医生复核、训练数据回流、随访等能力已建模，但演示深度低于分析与 RAG 主链。
