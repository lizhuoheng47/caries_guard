# 13. 比赛版环境与演示运行说明

## 1. 目标与受众

本文档专门为评审专家和答辩演示人员编写。
当前环境专为“一键复现式演示”剪裁，摒弃了所有复杂的依赖配置步骤。我们通过预置的 Docker 化方案和自动化探针，保障展示过程流畅稳定。

## 2. 默认环境预设

启动比赛版时，我们会自动挂载 `env/competition.env` 配置。核心隔离参数如下：
- **`CARIES_COMPETITION_MODE_ENABLED=true`**：开启比赛护栏，系统会自动屏蔽底层枯燥的管理页面，仅为您呈现 AI 分诊、智能读片和 RAG 结论。
- **`CG_AI_RUNTIME_MODE=mock`**：这是为了避免现场网络抖动或算力平台不稳定带来的故障。所有的 AI 回调将采用内置推理缓存进行仿真极速响应。

## 3. 标准演示启动流程

所有的生命周期管理脚本都存放在 `scripts/` 目录下。

### 3.1 启动系统
```powershell
./scripts/competition-up.ps1
```
*(Mac / Linux 用户执行对应的 `.sh` 脚本)*

该脚本会自动：
1. 拉起底座数据库与缓存。
2. 拉起 Java 与 Python 双引擎并等待探针探测联通。
3. 自动向数据层灌入 **Demo 患者与预训练知识库切片**。
当界面提示 `Competition Environment Successfully Started` 时，说明底层引擎已就绪。

### 3.2 登录验证
浏览器访问：http://localhost:8080
使用预置掩码医生账号登入：
- 用户名：`admin` (超级管理视角 / 会受 Competition Mode 过滤)
- 密码：`123456`

## 4. 故障转移与恢复 (Failover)

任何时候，如果您在展示中发现页面残留无关数据想重新开启一轮纯净审核，可以直接重置：
```powershell
./scripts/competition-reset.ps1
```
这会安全地切断容器并清空挂载的缓存卷册。此时重新执行 `competition-up`，您将得到与初始状态一模一样的 Demo 预设。
