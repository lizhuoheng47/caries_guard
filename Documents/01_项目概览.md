# 项目概览

## 1. 项目定位

CariesGuard 是一个面向龋病筛查场景的医疗 AI 辅助决策系统，核心主线固定为：

`影像分析 -> 不确定性评估 -> 医生复核 -> RAG 解释 -> 风险评估 -> 报告/随访`

系统目标不是替代医生诊断，而是提供可追踪、可解释、可复核的辅助结果。

## 2. 整体架构

```text
Client / Script
  -> Java Backend
      -> caries_biz (MySQL)
      -> Redis
      -> RabbitMQ
      -> MinIO
      -> Python Backend
          -> caries_ai (MySQL)
          -> Local Vector Index
          -> General LLM Provider
```

职责边界：

- Java 负责业务主链、权限、状态推进、报告/复核/随访、对外 API。
- Python 负责 AI 推理、RAG、知识库、模型运行治理、回调结果生成。
- MinIO 负责原始影像、可视化产物、报告、导出文件存储。
- RabbitMQ 负责分析任务异步解耦。

## 3. 核心链路

### 3.1 影像分析链路

1. Java 创建分析任务并写入 `ana_task_record`。
2. Java 将任务投递到 RabbitMQ。
3. Python 消费任务，下载影像，执行质量检查、检测、分割、分级、不确定性和风险流程。
4. Python 生成 visual assets，上传 MinIO。
5. Python 通过 callback 将结构化结果回传 Java。
6. Java 落库分析摘要、可视化附件、风险信息，并推进病例状态。

### 3.2 RAG 问答链路

1. Java 接收问答或解释请求。
2. Java 组装病例上下文并调用 Python RAG API。
3. Python 完成知识检索、引用组装、LLM 生成和安全约束。
4. Java 返回问答结果或写入业务结果页。

### 3.3 反馈闭环

1. 医生对分析结果进行复核或纠偏。
2. 纠偏数据进入 `ana_correction_feedback`。
3. 可导出为训练/评估数据快照。
4. 模型版本、评估记录、审批记录形成治理闭环。

## 4. 仓库结构

- `backend-java/`：Java 业务后端，多模块 Maven 工程。
- `backend-python/`：Python AI/RAG 服务，FastAPI + MQ Worker。
- `Documents/`：项目概览、功能说明、部署运行、接口集成、数据字典。
- `infra/`：数据库初始化脚本。
- `scripts/`：比赛演示、环境启动、验收和样例数据脚本。
- `env/`：预设环境变量文件。

## 5. 对外能力边界

- AI 输出是辅助信息，不是最终诊断结论。
- Python 不直接维护 `caries_biz` 业务状态。
- Java 不直接承担模型推理、向量检索和 LLM 生成。
- RAG 输出必须带引用、版本和安全约束。
- 高不确定性结果必须保留复核入口。

## 6. 运行模式

Python 运行模式：

- `mock`：全链路可复现演示模式，默认推荐。
- `hybrid`：部分模块启用真实适配器。
- `real`：所有启用模块必须真实执行，失败显式报错。

Java 比赛模式：

- `CARIES_COMPETITION_MODE_ENABLED=true`：收敛菜单、权限和页面暴露面。
- `CARIES_COMPETITION_MODE_ENABLED=false`：保留完整开发视角。
