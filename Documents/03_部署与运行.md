# 部署与运行

## 1. 运行依赖

根目录 `docker-compose.yml` 会统一启动以下组件：

- MySQL 8.4
- Redis 7
- RabbitMQ 3.13
- MinIO
- Java Backend
- Python Backend

默认采用双库：

- `caries_biz`
- `caries_ai`

## 2. 快速启动

在仓库根目录执行：

```powershell
docker compose up -d --build
```

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8080/actuator/health
Invoke-RestMethod http://127.0.0.1:8001/ai/v1/health
```

默认管理员账号：

- 用户名：`admin`
- 密码：`123456`

## 3. 比赛演示模式

仓库内置比赛演示预设文件 [env/competition.env](../env/competition.env)。

推荐命令：

```powershell
docker compose --env-file env/competition.env up -d --build
```

关键预设：

- `CARIES_COMPETITION_MODE_ENABLED=true`
- `CG_AI_RUNTIME_MODE=mock`
- `CG_RAG_KNOWLEDGE_VERSION=v1.0`
- `CG_LLM_PROVIDER_CODE=MOCK`

## 4. 标准脚本

推荐优先使用 `scripts/` 目录脚本：

- `competition-up.ps1` / `competition-up.sh`：启动演示环境
- `competition-acceptance.ps1` / `competition-acceptance.sh`：基础验收
- `competition-reset.ps1` / `competition-reset.sh`：演示后清理
- `phase5-analysis-docker-e2e.ps1`：影像分析闭环验证
- `phase3-rag-docker-e2e.ps1`：RAG 问答闭环验证
- `seed_demo_assets.*`：演示资源准备
- `seed_demo_data.sql`：演示数据灌库
- `seed_demo_knowledge.py`：演示知识库准备

## 6. 本地模块启动

### 6.1 Java

在 `backend-java/` 下：

```powershell
mvn -pl caries-boot -am -DskipTests compile
mvn -pl caries-boot -am spring-boot:run
```

### 6.2 Python

在 `backend-python/` 下：

```powershell
python -m app.main
```

本地分模块启动前，需要自行准备 MySQL、RabbitMQ、MinIO、Redis 及相应环境变量。仓库默认更推荐使用 Compose 联调。

## 7. 测试与验收

### 7.1 Java

示例：

```powershell
mvn -pl caries-system -am test -Dtest=AuthAppServiceTests,SystemAdminQueryAppServiceTests,SystemPermissionAuthorityServiceTests -Dsurefire.failIfNoSpecifiedTests=false
```

`backend-java/caries-boot/src/test/java/com/cariesguard/boot/e2e/` 下包含主链、看板、随访、模型治理等集成测试。

### 7.2 Python

示例：

```powershell
.\.venv\Scripts\python -m pytest tests\unit\test_rag_service.py tests\unit\test_inference_pipeline_phase5b.py tests\unit\test_schema_alias.py
```

`backend-python/tests/unit/` 已覆盖回调、推理流水线、模型适配器、RAG 服务、风险服务、存储和清理逻辑。

## 8. 存储与生命周期

MinIO 默认 bucket：

- `caries-image`：原始影像
- `caries-visual`：分割掩码、热力图等中间产物
- `caries-report`：报告文件
- `caries-export`：导出文件

生命周期约束：

- `caries-export` 默认 7 天过期
- `caries-visual` 默认 30 天过期

## 9. 常见运行建议

- 需要稳定演示时，优先使用 `mock`。
- 需要证明真实适配能力时，再切换到 `hybrid`。
- `real` 模式仅建议在依赖、模型和回调链路均已确认时使用。
- 文档、脚本和环境变量应始终以根目录 `docker-compose.yml` 为准。
