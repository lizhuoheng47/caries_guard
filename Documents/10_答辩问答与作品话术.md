# 答辩问答与作品话术

## 1. 推荐答辩主线

1. 龋病筛查场景有什么问题
2. 单点模型为什么不够
3. 系统如何形成 AI 闭环
4. 如何控制医疗 AI 风险
5. 如何证明系统有效
6. 方案的落地价值是什么

## 2. 建议统一话术

### 作品定位

“我们做的不是普通口腔管理平台，而是一个面向龋病筛查场景的医疗 AI 辅助决策闭环系统。”

### 关于影像分析

“系统不是黑盒直接出结论，而是经过质量评估、检测、分割、分级和 uncertainty 分析后输出结构化结果。”

### 关于 uncertainty

“我们不会盲目信任模型，高 uncertainty 样本会自动进入医生复核流程。”

### 关于 RAG

“我们不是让大模型自由发挥，而是让它在权威知识和病例上下文约束下生成解释，并返回 citations 与 safety flags。”

### 关于反馈闭环

“医生可以修正 AI 结果，修正后的差异会被记录下来，用于一致率统计、误差分析和后续治理。”

## 3. 常见问题与回答

### Q1: 为什么采用 Java + Python 双后端？

A: Java 负责业务主链、状态权威和系统协同，Python 负责 AI/RAG 能力，职责清晰，便于扩展和治理。

### Q2: 为什么不直接用大模型给诊断？

A: 医疗场景需要严格边界。大模型用于解释与辅助，不替代医生最终诊断。

### Q3: 你们的创新点是什么？

A:

- 影像分析与知识增强协同
- uncertainty 驱动的人机协作
- 带引用与安全约束的 RAG
- 医生反馈回流闭环
- 双后端、双库、可审计留痕的工程实现

### Q4: 如何证明系统可信？

A:

- 有 visual assets 和 raw JSON 留痕
- 有 citations 和 retrieved chunks
- 有 uncertainty 复核机制
- 有反馈统计和评估指标

### Q5: 如何落地？

A: 当前架构已经具备异步任务、对象存储、消息队列、运行日志和知识治理能力，具备继续工程化落地的基础。

## 4. 结束总结词

“我们的重点不是做很多页面，而是把影像分析、知识解释、风险建议和医生反馈组织成一个可解释、可追踪、可治理的医疗 AI 闭环。”
