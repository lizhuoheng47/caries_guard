# 知识图库建设与治理规范

## 1. 目标

将知识库从“附属问答数据”提升为 RAG 的事实约束中枢。

## 2. 知识来源分层

1. 权威指南层
2. 科普说明层
3. 业务规则层
4. 案例经验层

## 3. 入库字段

每份文档至少记录：

- `kbCode`
- `docTitle`
- `docSourceCode`
- `sourceUri`
- `docVersion`
- `reviewStatusCode`
- `knowledgeVersion`

## 4. 治理要求

- 未审核文档不得进入正式检索范围
- 文档版本必须可追溯
- chunk 必须可关联到原始文档
- 检索日志必须保留
- 回答必须带 citations
- 知识失效后必须支持人工下线

## 5. 检索输出要求

RAG 响应至少返回：

- `answer`
- `citations`
- `retrievedChunks`
- `knowledgeBaseCode`
- `knowledgeVersion`
- `safetyFlags`
- `refusalReason`

## 6. 运维指标

- 知识命中率
- 被引用文档分布
- chunk 命中频次
- 失效文档占比
- 审核通过率

## 7. 当前仓库对应能力

- Python 侧知识文档导入
- chunk rebuild
- 向量检索
- `rag_request_log`
- `rag_retrieval_log`
- `llm_call_log`

## 8. 后续建议

- 增加知识下线操作规范
- 增加知识命中统计报表
- 建立高频问答的引用质量抽检机制
