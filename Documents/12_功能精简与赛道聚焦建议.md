# 功能精简与赛道聚焦建议

## 1. 结论

需要精简，但不是删库重做，而是做三层收束：

1. 竞赛叙事精简
2. 演示路径精简
3. 工程暴露面精简

目标是把完整平台收束成一个强聚焦的 AI 闭环作品。

## 2. 当前仓库的主要矛盾

### 2.1 Java 侧业务域覆盖很宽

已经覆盖 system、patient、image、analysis、report、followup、dashboard 等多个业务域，工程骨架完整。

### 2.2 Python 侧当前仍然是 AI 能力底座

虽然已经打通影像分析和 RAG 最小闭环，但当前最强项仍然是工程骨架，不是一个大而全的 AI 成果展示。

### 2.3 风险

如果继续按完整业务平台包装，评委容易认为：

- 这是一个医疗后台系统
- AI 只是系统里的一个子功能

## 3. 必须保留并突出展示的能力

### 3.1 影像分析主链

- 病例/影像提交
- analysis task 创建
- MQ 投递
- Python 消费
- quality / detection / segmentation / grading / uncertainty
- callback 回写
- 结果落库与状态推进

### 3.2 可视化结果

- MASK
- OVERLAY
- HEATMAP
- gradingLabel
- uncertaintyScore
- needsReview

### 3.3 RAG 智能解释

- 知识入库
- 索引重建
- 患者解释
- 医生问答
- citations
- retrieval / llm / request 日志

### 3.4 医生复核与反馈

- 高 uncertainty 进入复核
- 医生修正结果
- AI 与医生差异留痕
- 反馈回流到评估和优化链路

### 3.5 最小报告生成

仅保留结果汇总、解释展示和复核状态，不强调复杂模板系统。

## 4. 建议压缩的功能

### 4.1 system 模块

压缩展示，不作为主卖点：

- 用户管理
- 角色管理
- 菜单管理
- 字典管理
- 配置管理
- 数据权限规则管理

### 4.2 patient 模块

保留“患者 - 病例 - 影像”最小关系，压缩复杂监护人与多层就诊细节。

### 4.3 report 模块

保留最小结果页，压缩模板管理与导出审计。

### 4.4 followup 模块

建议从参赛主舞台移除，只保留代码与后续扩展能力，不进入主演示链路。

### 4.5 dashboard 模块

只保留 AI 指标：

- 模型运行量
- 不确定性分布
- 复核率
- 一致率
- RAG 调用量

不建议展示通用 BI 总览。

## 5. 建议隐藏或延后的能力

- 菜单/权限/组织多租户细节
- LOCAL_FS 等调试型存储路线
- Redis 作为非主亮点基础设施
- 过宽的 RAG 表述

## 6. 推荐的参赛版功能边界

### 模块 1：病例与影像输入

- 新建病例
- 上传影像
- 查看病例上下文

### 模块 2：AI 分析任务

- 创建任务
- 查看任务状态
- 回调结果写回

### 模块 3：AI 结果展示

- 分级结果
- 不确定性
- 可视化产物
- 风险辅助信息

### 模块 4：智能解释

- 患者版解释
- 医生版问答
- 引用与安全提示

### 模块 5：人工复核与反馈

- 触发复核
- 医生修正
- 差异记录
- 闭环留痕

### 模块 6：AI 评估看板

- uncertainty 分布
- 复核率
- 一致率
- RAG 引用准确率

## 7. 工程建议

### 第一优先

先精简演示，不先动底层代码：

- 精简菜单
- 精简入口
- 精简说明书
- 精简答辩 PPT

### 第二优先

增加“竞赛模式”开关：

- 隐藏 system / followup / 复杂 report 菜单
- 默认展示 AI 主链
- 固定演示路径和演示账号

### 第三优先

按页面和权限暴露收缩，而不是贸然删除整个模块。

### 第四优先

精简演示数据，只保留：

- 患者
- 病例
- 影像
- 分析任务
- 分析结果摘要
- visual asset
- 复核记录
- RAG 知识文档
- RAG 请求日志
- 医生修正反馈

## 8. 不应精简的部分

- uncertainty 机制
- visual asset
- callback 契约与结果留痕
- 医生修正反馈
- RAG 的引用与日志

## 9. 一句话策略

当前项目的问题不是功能太少，而是功能面太宽、AI 主线不够突出。

最优策略不是删库重做，而是把完整平台收束成一个强聚焦的 AI 闭环作品。
