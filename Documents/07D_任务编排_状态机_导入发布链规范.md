# 07D 任务编排、状态机、导入发布链规范

## 1. 文档状态机

```text
DRAFT -> REVIEW_PENDING -> APPROVED -> PUBLISHED
APPROVED -> REJECTED
PUBLISHED -> ROLLED_BACK -> PUBLISHED(历史版本重新发布)
```

固定规则：

1. 编辑已发布内容时必须创建新 `kb_document_version`。
2. 一个文档同一时刻只允许一个 `published_version_no`。

## 2. ingest job 状态机

```text
PENDING -> RUNNING -> SUCCESS
PENDING -> RUNNING -> FAILED
FAILED -> RUNNING  (允许人工重试时)
```

固定 step 顺序：

1. `PARSE`
2. `NORMALIZE`
3. `CHUNK`
4. `INDEX`
5. `GRAPH_SYNC`
6. `SUBMIT_REVIEW`

## 3. graph sync 状态机

```text
SYNC_REQUESTED -> SUCCESS
SYNC_REQUESTED -> FAILED
```

固定补偿：

1. 失败后写 `kb_graph_sync_log.error_message`。
2. rebuild 时允许重新触发图谱同步。

## 4. review / publish / rollback 规则

| 动作 | 前置条件 | 落库 | 失败后执行 |
| --- | --- | --- | --- |
| `submit-review` | 版本存在 | `kb_review_record` | 保持 `DRAFT` |
| `approve` | `REVIEW_PENDING` | `kb_review_record` | 版本不进入发布 |
| `reject` | `REVIEW_PENDING` | `kb_review_record` | 版本维持不可发布 |
| `publish` | 已审核通过 | `kb_publish_record` | 不更新 `published_version_no` |
| `rollback` | 历史版本存在 | `kb_publish_record` | 保持当前 published 版本 |

## 5. rebuild job 规则

| 项目 | 固定策略 |
| --- | --- |
| 输入 | 已发布文档版本 |
| 同步动作 | OpenSearch 索引删除与写入、Neo4j 清理与重建 |
| 异步动作 | 可由后台任务触发，结果落 `kb_rebuild_job` |
| 幂等键 | `rebuild_job_no` |
| 失败补偿 | 保留旧 published 版本；失败 job 单独记录 |

## 6. 幂等与重试

1. `source_file_no`、`ingest_job_no`、`run_no` 必须唯一。
2. 图谱重建先清理同文档数据，再执行 `MERGE`。
3. Java 到 Python 的状态动作必须带 `traceId`。
4. 回调与状态推进必须容忍重复请求，不允许重复生成新状态。

## 7. 手工介入点

| 场景 | 介入点 |
| --- | --- |
| 解析失败 | 重新上传或修正文档内容后重试 |
| 图谱抽取偏差 | 在审核阶段阻断发布 |
| 检索效果下降 | 触发 rebuild 并比对评估结果 |
| 回答越权 | 通过日志定位 requestNo / traceId 后修正规则 |
