# 03 业务主链路与状态机说明

## 1. Analysis 主链路

```text
create task
  -> PENDING
  -> publish MQ
  -> RUNNING
  -> Python callback
  -> SUCCESS / FAILED
```

## 2. 成功路径

成功 callback 后 Java：

- 更新 `ana_task_record`；
- 写 `ana_result_summary`；
- 写 `ana_visual_asset`；
- 保存 visual 附件；
- 根据 `needsReview` 判断复核流转；
- 支撑 report/followup 后续链路。

## 3. 失败路径

失败 callback 后 Java：

- 写 `error_code`；
- 写 `error_message`；
- 状态置为 `FAILED`；
- 不伪造成功 summary。

`real` 模式下 Python 失败必须显式失败，不允许静默回退。

## 4. Review 语义

```text
needsReview=false -> ReportReady
needsReview=true  -> ReviewPending
```

复核结果可以确认或修正 AI 建议，但必须保留 AI 原始证据。

## 5. RAG 主链路

RAG 是辅助解释链路：

```text
question -> Java auth/context -> Python RAG -> answer/citations -> display
```

RAG 结果不直接改变 task、report、review 状态。

## 6. 通用大模型边界

通用大模型只负责文本生成和解释。事实依据来自知识图库，业务状态来自 Java 规则，医疗结论来自医生确认。
