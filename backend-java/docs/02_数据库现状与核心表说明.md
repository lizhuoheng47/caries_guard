## 4. V014 重点结构

### 4.1 `ana_task_record`

新增：

| 字段 | 说明 |
| --- | --- |
| `trace_id` | Python AI 服务 trace ID |
| `inference_millis` | 推理耗时，毫秒 |

### 4.2 `ana_correction_feedback`

新增：

| 字段 | 说明 |
| --- | --- |
| `training_candidate_flag` | 是否训练候选 |
| `desensitized_export_flag` | 是否脱敏导出 |
| `dataset_snapshot_no` | 数据集快照编号 |
| `review_status_code` | 审核状态 |
| `reviewed_by` | 审核人 |
| `reviewed_at` | 审核时间 |

### 4.3 `ana_model_version_registry`

用途：模型版本登记、审批状态和版本状态追踪。当前不等于完整独立 ModelAdmin 平台。

字段：`model_code`、`model_version`、`model_type_code`、`approved_flag`、`approved_by`、`approved_at`、`status`、`remark`。

## 5. 对象存储数据口径

`med_attachment.storage_provider_code` 当前可能值：

| 值 | 说明 |
| --- | --- |
| `MINIO` | 默认实现，文件在 MinIO |
| `LOCAL_FS` | e2e/兼容实现，文件在本地目录 |

Python 联调优先使用 `AiAnalysisRequestDTO.images[].accessUrl`，不要把 `object_key` 当成本地路径。

## 6. 数据治理边界

1. 业务库已覆盖平台闭环。
2. 训练候选治理已有字段落点。
3. 标注平台、训练快照平台、完整 ModelAdmin 平台未独立实现。
4. 风险评估能力由 `med_risk_assessment_record` 承载，不是独立模块。
