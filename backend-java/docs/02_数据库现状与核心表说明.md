# 数据库现状与核心表说明

更新日期：2026-04-16

## 1. 当前基线

| 项 | 当前值 |
| --- | --- |
| Flyway 基线 | `V001__baseline_schema.sql` |
| 迁移文件数量 | 1 |
| 总表数 | 38 |
| 默认对象存储 | `MINIO` |
| MinIO 基础设施实现 | `caries-integration` 的 `MinioObjectStorageClient` |

历史多文件迁移已压缩进单基线。新环境只执行 `V001__baseline_schema.sql`；已有旧 `flyway_schema_history` 的本地库需要清库重建后再启动。

## 2. 基线包含内容

| 数据域 | 内容 |
| --- | --- |
| 系统管理 | 用户、角色、菜单、数据权限、字典、登录日志、操作日志 |
| 患者病例 | 患者、监护人、档案、就诊、病例、牙位、状态流转 |
| 影像与附件 | `med_attachment`、`med_image_file`、`med_image_quality_check` |
| AI 分析 | 任务、摘要、visual asset、医生修正、模型版本登记 |
| 报告 | 报告记录、模板、导出日志 |
| 随访 | 随访计划、任务、记录 |
| 看板与治理 | 风险记录、模型运行统计所需字段、训练候选治理字段 |

## 4. 重点治理字段

| 对象 | 字段/内容 |
| --- | --- |
| `ana_task_record` | `trace_id`、`inference_millis`、`request_batch_no`、`callback_payload_json`、`error_code` |
| `ana_result_summary` | `overall_highest_severity`、`uncertainty_score`、`review_suggested_flag`、`lesion_count`、`abnormal_tooth_count`、`summary_version_no` |
| `ana_visual_asset` | `related_image_id`、`source_attachment_id`、`tooth_code`、`sort_order` |
| `ana_correction_feedback` | `training_candidate_flag`、`desensitized_export_flag`、`dataset_snapshot_no`、`review_status_code`、`source_attachment_id`、`export_candidate_flag`、`exported_snapshot_no` |
| `med_attachment` | `asset_type_code`、`source_attachment_id`、`retention_policy_code`、`expired_at`、`integrity_status_code`、`metadata_json` |

## 5. 对象存储数据口径

`med_attachment.storage_provider_code` 当前可能值：

| 值 | 说明 |
| --- | --- |
| `MINIO` | 默认实现，文件在 MinIO |
| `LOCAL_FS` | 不再作为当前 Java 运行口径；历史数据或测试桩中可能出现 |

Python 联调优先使用 `AiAnalysisRequestDTO.images[].accessUrl` 拉取原图；可视化结果由 Python 上传到 `caries-visual` 后回调 `bucketName + objectKey`。

## 6. 数据治理边界

1. 业务库已覆盖平台闭环。
2. 训练候选治理已有字段落点。
3. 标注平台、训练快照平台、完整 ModelAdmin 平台未独立实现。
4. 风险评估能力由 `med_risk_assessment_record` 承载，不是独立模块。
