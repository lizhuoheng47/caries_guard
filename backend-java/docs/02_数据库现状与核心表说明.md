# 02 数据库现状与核心表说明

## 1. Java 数据库

Java 使用 `caries_biz`，只保存业务主数据。

## 2. 核心表

| 表 | 说明 |
| --- | --- |
| `pat_patient` | 患者 |
| `med_visit` | 就诊 |
| `med_case` | 病例 |
| `med_attachment` | 附件 |
| `med_image_file` | 影像 |
| `ana_task_record` | 分析任务 |
| `ana_result_summary` | 分析摘要和 AI 原始结果 |
| `ana_visual_asset` | visual asset 关联 |
| `rpt_report` | 报告 |
| `rev_review_record` | 复核 |
| `fup_followup_plan` | 随访 |

## 3. 与 Python 数据库关系

Python 使用 `caries_ai` 记录推理运行、RAG、通用大模型调用和模型/知识治理。

Java 与 Python 通过 `task_no`、`image_id`、`trace_id` 弱引用，不建立跨库外键。

## 4. AI 原始结果

`ana_result_summary.raw_result_json` 是 Java 消费 AI 扩展字段的主要位置。

Phase 5C 必备字段：

- `gradingMode`
- `gradingImplType`
- `gradingLabel`
- `confidenceScore`
- `uncertaintyMode`
- `uncertaintyImplType`
- `uncertaintyScore`
- `needsReview`

## 5. RAG 数据

知识库、chunk、retrieval log、llm call log 均归属 Python `caries_ai`。Java 只保存业务展示需要的记录。
