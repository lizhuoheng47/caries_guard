# 运行测试与已知边界

本文档说明当前项目如何启动、如何验证，以及哪些限制已经确认存在。

## 1. 启动方式

### 1.1 基本环境

本地 profile 依赖：
- MySQL
- Redis
- RabbitMQ

关键配置位置：
- `caries-boot/src/main/resources/application.yml`
- `caries-boot/src/main/resources/application-local.yml`
- `caries-boot/src/main/resources/application-e2e.yml`

### 1.2 启动模块

应用启动入口：
- `caries-boot`

启动类：
- `com.cariesguard.boot.CariesBootApplication`

常用启动命令：

```bash
mvn -pl caries-boot spring-boot:run -Dspring-boot.run.profiles=local
```

或：

```bash
mvn -pl caries-boot -am package
java -jar caries-boot/target/*.jar --spring.profiles.active=local
```

## 2. 访问入口

本地服务地址：
- `http://127.0.0.1:8080`

常用地址：
- Swagger UI: `http://127.0.0.1:8080/swagger-ui/index.html`
- OpenAPI: `http://127.0.0.1:8080/v3/api-docs`
- Health: `http://127.0.0.1:8080/actuator/health`
- Ping: `http://127.0.0.1:8080/api/v1/system/ping`

## 3. 当前数据库初始化结论

初始化来源：
- Flyway 自动执行 `db/migration` 下迁移脚本

当前结论：
- 结构已经到 `V013`
- 默认只含系统种子数据
- 默认没有业务样例数据

这意味着：
- 登录能直接测
- 系统元数据能直接测
- 业务链路需要先造患者、就诊、病例、图像等数据

## 5. 当前运行时限制

### 5.1 Rabbit 依赖

当前 local profile 默认是 `rabbit` 模式，因此：
- 没有 RabbitMQ 时，分析任务事件发布会受影响
- 没有 Python 消费端时，任务会停在等待外部处理的阶段

### 5.2 对象存储实现与配置名不一致

当前事实：
- local profile 默认 `provider-code=MINIO`
- `MINIO` 使用 `MinioObjectStorageService`，`LOCAL_FS` 使用 `LocalObjectStorageService`

影响：
- 文档可以写“Java 后端已接入 MinIO provider”，但不能写“仓库内已包含完整 MinIO 部署编排”
- Python 侧也不能假设已经有标准对象存储 HTTP 接口

### 5.3 PDF 生成能力有限

当前 `ReportPdfService` 仅用于：
- 证明流程闭环
- 产出简单 PDF 附件

不适合：
- 复杂排版
- 中文完整渲染
- 生产级报告出图

### 5.4 导出接口不是下载接口

`POST /api/v1/reports/{reportId}/export` 当前只是：
- 校验报告
- 写导出日志
- 返回导出成功标志

当前不是：
- 文件下载接口
- 外发通道接口

### 5.5 权限演示不完整

Flyway V014 已补充 `sys_menu` 和 `sys_role_menu` 种子数据：
- 普通角色的基础菜单权限场景可在执行 V014 后演示
- 默认管理员仍可全量联调，ORG_ADMIN/DOCTOR/SCREENER 可用于基础权限演示

## 6. 当前适合做什么

当前项目最适合的用途：
- 后端主链路联调
- Python AI 回调对接
- 业务闭环演示
- 课程/答辩/阶段验收
- 作为 Python 侧重写或复刻业务的参考样板

## 7. 当前不应误判为已经完成的事项

1. 不能认为仓库内已经包含完整 MinIO 服务编排；Java 后端已经有 MinIO provider，运行环境仍需提供 MinIO 服务。
2. 不能认为已经有 Python 消费端源码。
3. 报告导出已经返回签名下载 URL；仍需由对象存储服务和 `/api/v1/files/{attachmentId}/content` 保证实际下载链路可访问。
4. 普通角色权限演示依赖 Flyway V014 菜单/角色/数据权限种子，执行迁移后可演示基础授权。
5. 不能认为默认数据库已经自带可展示的业务数据。

## 8. 若要继续扩展，最自然的下一步

按优先级建议：
1. 补业务样例数据和菜单种子数据
2. 给 Python 消费端补一个标准示例工程
3. 继续完善 Python 消费端，并基于当前 `accessUrl/accessExpireAt` 载荷完成端到端拉图验证
4. 替换 `ReportPdfService` 为正式 PDF 渲染方案
5. 把数据权限规则真正下推到各业务模块
