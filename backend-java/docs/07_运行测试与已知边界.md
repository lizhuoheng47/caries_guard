# 运行测试与已知边界

更新日期：2026-04-15

## 1. 运行前提

| 依赖 | 说明 |
| --- | --- |
| JDK | Java 17+ |
| Maven | 用于编译和测试 |
| MySQL | local profile 业务库 |
| RabbitMQ | 分析任务消息 |
| MinIO | local 默认对象存储 |
| Python AI 服务 | 仓库外实现，用于完整联调 |

## 2. 常用命令

```powershell
cd E:\caries_guard\backend-java
mvn -q -DskipTests compile
mvn -q test
mvn -q -pl caries-analysis -am test
```

## 3. 对象存储运行模式

| 模式 | 配置 | 说明 |
| --- | --- | --- |
| MinIO | `CARIES_IMAGE_STORAGE_PROVIDER=MINIO` | local 默认，需要启动 MinIO |
| LocalFS | `CARIES_IMAGE_STORAGE_PROVIDER=LOCAL_FS` | e2e/临时本地使用 |

MinIO 环境变量示例：

```powershell
$env:CARIES_IMAGE_STORAGE_PROVIDER='MINIO'
$env:CARIES_MINIO_ENDPOINT='http://127.0.0.1:9000'
$env:CARIES_MINIO_ACCESS_KEY='minioadmin'
$env:CARIES_MINIO_SECRET_KEY='minioadmin'
$env:CARIES_IMAGE_PUBLIC_BASE_URL='http://127.0.0.1:8080'
```

## 4. 数据库

1. Flyway 结构已经到 V014。
2. V014 包含模型版本登记、AI trace/耗时、训练准入字段、业务角色、菜单和数据权限种子。
3. 默认业务样例数据仍较少，业务演示需要通过 API 创建患者、就诊、病例、影像、分析、报告和随访数据。
4. 当前系统种子包含管理员、业务角色、菜单和基础数据权限规则。

## 5. PDF 字体

`ReportPdfService` 使用 PDFBox，并按候选路径加载中文字体。正式演示环境建议安装以下任一字体：

1. `NotoSansSC`
2. `SimHei`
3. `Simsun`
4. Linux 下 Noto CJK 或文泉驿字体

若没有中文字体，PDFBox 会降级，无法编码字符会替换为 `?`。

## 6. 当前边界

1. Java 代码已经实现 MinIO provider，但不会自动启动 MinIO 服务。
2. Python AI 服务源码不在当前仓库，需要按 `06_Python联调与AI协作说明.md` 对接。
3. 前端源码不在当前仓库。
4. ModelAdmin 没有独立模块，只有最小模型治理落点。
5. Risk 没有独立模块，风险评估由分析结果和风险记录表承载。
6. 数据权限规则已有种子，但业务查询层后续仍可继续深化 `ALL/ORG/DEPT/SELF/CUSTOM` 下推。
7. 标注平台和训练数据快照平台没有独立实现。

## 7. 推荐验收路径

1. 启动 MySQL、RabbitMQ、MinIO。
2. 启动 Java local profile。
3. 登录管理员账号。
4. 创建患者、就诊、病例。
5. 上传影像，确认 `med_attachment.storage_provider_code=MINIO`。
6. 创建分析任务，确认消息包含 `accessUrl`。
7. 用 Python 或模拟请求回调 `AiAnalysisResultCallbackCommand`。
8. 查看任务详情、dashboard `model-runtime`、风险分布。
9. 生成报告并调用导出接口，确认返回 `downloadUrl`。
10. 创建随访计划、任务和记录。
