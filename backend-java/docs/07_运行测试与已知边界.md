# 运行测试与已知边界

更新日期：2026-04-15

## 1. 运行前提

| 依赖 | 说明 |
| --- | --- |
| JDK | Java 17+ |
| Maven | 用于编译和测试 |
| MySQL | local profile 业务库 |
| RabbitMQ | 分析任务消息 |
| MinIO | local/docker 默认对象存储 |
| Python AI 服务 | `backend-python` 已有骨架，真实模型逻辑待实现 |

## 2. 常用命令

```powershell
cd E:\caries_guard\backend-java
mvn -q -DskipTests compile
mvn -q test
mvn -q -pl caries-analysis -am test
```

完整测试包含 Boot E2E，启动 Spring/MySQL 较慢，当前本机完整 `mvn -q test` 约 3 分钟。

## 3. 对象存储运行模式

| 模式 | 配置 | 说明 |
| --- | --- | --- |
| MinIO | `CARIES_STORAGE_PROVIDER=MINIO` | 当前唯一长期运行口径 |

MinIO 环境变量示例：

```powershell
$env:CARIES_STORAGE_PROVIDER='MINIO'
$env:CARIES_MINIO_ENDPOINT='http://127.0.0.1:9000'
$env:CARIES_MINIO_ACCESS_KEY='minioadmin'
$env:CARIES_MINIO_SECRET_KEY='minioadmin'
$env:CARIES_BUCKET_IMAGE='caries-image'
$env:CARIES_BUCKET_VISUAL='caries-visual'
$env:CARIES_BUCKET_REPORT='caries-report'
$env:CARIES_BUCKET_EXPORT='caries-export'
```

## 5. PDF 字体

`ReportPdfService` 使用 PDFBox，并按候选路径加载中文字体。正式演示环境建议安装以下任一字体：

1. `NotoSansSC`
2. `SimHei`
3. `Simsun`
4. Linux 下 Noto CJK 或文泉驿字体

若没有中文字体，PDFBox 会降级，无法编码字符会替换为 `?`。

## 6. 当前边界

1. Java 代码已经实现 MinIO provider，但不会自动启动 MinIO 服务；本地需要手动启动，Docker Compose 会启动。
2. `caries.storage.auto-create-buckets` 在 local 默认为 `true`，Docker 默认为 `false`；Docker 环境依赖 `minio-init` 创建四个 bucket 和生命周期规则。
3. Python AI 服务骨架在 `backend-python`，真实推理、visual asset 上传和模型加载仍需继续实现。
4. 前端源码不在当前仓库。
5. ModelAdmin 没有独立模块，只有最小模型治理落点。
6. Risk 没有独立模块，风险评估由分析结果和风险记录表承载。
7. 数据权限规则已有种子，但业务查询层后续仍可继续深化 `ALL/ORG/DEPT/SELF/CUSTOM` 下推。
8. 标注平台和训练数据快照平台没有独立实现。

## 7. 推荐验收路径

1. 启动 MySQL、RabbitMQ、MinIO。
2. 启动 Java local profile。
3. 登录管理员账号。
4. 创建患者、就诊、病例。
5. 上传影像，确认 `med_attachment.bucket_name=caries-image`、`storage_provider_code=MINIO`。
6. 创建分析任务，确认消息包含 MinIO 预签名 `accessUrl`。
7. 用 Python 或模拟请求回调 `AiAnalysisResultCallbackCommand`，visual asset 可回传 `caries-visual + objectKey`。
8. 查看任务详情、dashboard `model-runtime`、风险分布。
9. 生成报告，确认报告附件在 `caries-report`。
10. 调用导出接口，确认导出附件在 `caries-export` 且返回 `downloadUrl`。
11. 创建随访计划、任务和记录。
