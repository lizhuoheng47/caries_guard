# 07 运行测试与已知边界

## 1. 本地测试

Java 单元测试按 Maven 模块执行。Docker E2E 用根目录脚本执行。

## 2. Docker

```powershell
docker compose up -d --build
docker compose ps
```

## 3. Phase 5C E2E

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\phase5-analysis-docker-e2e.ps1 -SkipComposeUp -Phase5COnly -WaitSeconds 180
```

通过标准：

- mock grading 成功；
- hybrid low uncertainty 成功且 `needsReview=false`；
- hybrid high uncertainty 成功且 `needsReview=true`；
- real grading failure 失败且错误码明确；
- Java callback 不破；
- visual assets 正常落库。

## 4. 已知边界

- 当前影像模型为算法/规则适配器，不是最终医学模型；
- 通用大模型输出只作辅助说明；
- RAG 依赖知识库质量；
- 不做业务专用大模型微调；
- 医生复核仍是高不确定性结果的关键闭环。

## 5. 回归要求

每次修改以下内容必须跑 E2E：

- callback DTO；
- rawResultJson 字段；
- MinIO visual asset；
- grading/uncertainty；
- RAG response 结构；
- 数据库 migration。
