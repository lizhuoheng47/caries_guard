## 3. 当前剩余计划

- 补齐 `sys_menu`、`sys_role_menu`、普通角色与权限码种子数据

### 3.2 数据权限

### 3.3 报告产品化

- 引入支持中文与复杂布局的 PDF 渲染方案
- 让模板版本真正可追踪
- 补齐导出下载链路

### 3.4 基础设施产品化

- 评估对象存储切换到 MinIO / OSS / S3
- 明确 RabbitMQ 在线联调方式
- 视需要把 Redis 纳入真实业务链路

## 4. 当前不应继续做的事

- 不应再扩散新的“未来大蓝图”文档
- 不应在普通角色权限未补齐前继续新增大量后台管理接口
- 不应在 PDF / 存储 / 部署未稳定前过早宣称生产可用

## 5. 当前计划结论

当前项目的正确推进方向不是再补主功能，而是做交付质量和工程化收口。
