# Schema Gap Analysis

## 1. 当前目的

本文记录“代码、Flyway、当前文档”之间仍然存在的剩余差距。注意，这些差距已经远小于旧版文档时期。

## 2. 当前已对齐的部分

- 核心业务表已经存在
- image 质检表已入迁移
- analysis 聚合列和重试血缘字段已入迁移
- followup 计划/任务/记录字段已经与代码对齐
- 报告模板与导出审计表已落地

## 3. 当前剩余差距

### 3.1 权限数据差距

- schema 有 `sys_menu`、`sys_role_menu`
- 代码有权限切面与查询逻辑
- 当前 `cg` 默认库缺少菜单种子数据

### 3.2 数据权限差距

- schema 有 `sys_data_permission_rule`
- system 模块有服务和仓储
- 业务模块查询尚未全面按规则裁剪

### 3.3 对象存储差距

- schema 和代码都支持附件抽象
- 配置上保留云存储语义
- 实际实现仍是本地文件系统

### 3.4 报告导出差距

- schema 有 `rpt_export_log`
- 代码会写导出审计
- 但没有独立导出文件生成/下载封装

### 3.5 PDF 能力差距

- 报告主链路存在
- 但生成器仍是最小化 PDF 实现

## 4. 当前结论

当前 schema 已经不再是“设计超前、代码滞后”的主要瓶颈。

更真实的瓶颈变成了：

- 种子数据
- 部署外部依赖
- 权限演示完整性
- 报告输出产品化
