# 测试验收与里程碑交付标准

## 1. 当前阶段结论

当前项目已经不是“先写验收标准”，而是已有真实测试资产。本文改为记录当前可用的验收证据与仍未覆盖的测试缺口。

## 2. 当前测试分层

### 2.1 模块单测

已明显存在单测的模块：

- `caries-system`
- `caries-analysis`

### 2.2 boot 真库测试

`caries-boot` 中已经存在以下类型的真库测试：

- 主链路 E2E
- analysis -> report E2E
- analysis -> report -> followup E2E
- analysis 回调幂等 E2E
- followup 审计 / 逾期 / 幂等测试
- dashboard 七个接口集成测试
- report 导出审计测试

### 2.3 E2E 独立 profile

`application-e2e.yml` 当前特点：

- 连接 `cg_e2e`
- Flyway 允许 clean
- analysis messaging mode = `logging`
- rabbit health disabled
- 本地对象存储目录切到 `var/image-storage-e2e`

- 登录成功/失败

## 5. 当前阶段的验收定义

在当前仓库中，“功能完成”至少应满足：

- 控制器可调用
- 数据库表结构已由 Flyway 固化
- 关键写路径有审计或状态日志
- 至少有模块单测或 boot 真库测试覆盖
- 文档与代码口径一致

## 6. 当前尚未形成充分证据的部分

- Redis 相关能力没有独立测试证据，因为业务尚未真正使用
- 普通角色权限矩阵缺少真实菜单种子数据，难以形成完整非管理员权限验收
- 生产级 PDF 效果、中文字体、数字签名没有自动化验收
- RabbitMQ 实时联调依赖外部环境，仓库内 E2E 默认切换到 logging 模式
