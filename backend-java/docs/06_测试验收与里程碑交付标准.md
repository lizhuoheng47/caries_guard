# 测试验收与里程碑交付标准

> 文档定位：后端开发验收标准  
> 目标：把“开发完成”定义为可测试、可验证、可演示  
> 适用范围：P0 ~ P8 全阶段

---

# 1. 总体原则

1. 没有测试的功能，不算真正完成；
2. 没有验收口径的模块，不允许口头宣布完成；
3. 没有文档同步的数据库变更，不允许合并；
4. 没有至少一条联调链路的模块，不算具备演示价值；
5. 国一标准下，必须拿得出稳定主链路演示与审计证据。

---

# 2. 分阶段里程碑验收

## P0 基线对齐
### 验收标准
- README、架构、计划、gap 文档齐全；
- V1 正式表范围冻结；
- Flyway 基线无冲突；
- 缺失表/索引/约束形成迁移方案；
- 本地空库可成功执行迁移。

## P1 system 完成
### 验收标准
- 登录成功；
- `/me` 可用；
- 用户/角色/菜单/字典基础接口可用；
- 登录日志落库；
- 基础操作日志可落库；
- org-aware 查询模式初步成立；
- 至少完成 10 个以上 system 模块单元/集成测试。

## P2 patient + case 完成
### 验收标准
- 患者建档成功；
- 就诊创建成功；
- 病例创建成功；
- 病例分页与详情可查；
- 病例状态机生效；
- 非法状态跳转被阻断；
- `med_case_status_log` 完整留痕。

## P3 image 完成
### 验收标准
- 附件上传成功；
- 影像关联成功；
- `med_image_quality_check` 可写可查；
- 主图规则生效；
- attachment 与 image 关系一致；
- 对象存储访问可用。

## P4 analysis 完成
### 验收标准
- 可创建 AI 任务；
- 任务状态可追踪；
- AI 成功回调可写入摘要与资产；
- AI 失败回调可记录异常；
- 回调幂等；
- 病例状态同步正确。

## P5 report 完成
### 验收标准
- 同一病例可生成医生版与患者版报告；
- 报告版本不可覆盖；
- 导出可审计；
- 报告列表与详情可查。

## P6 followup 完成
### 验收标准
- 可从报告/风险结果触发随访；
- 随访计划、任务、记录可查；
- 通知发送留痕；
- 随访完成可回写病例闭环。

## P7 dashboard 完成
### 验收标准
- 至少完成病例数、风险分布、随访状态等核心聚合接口；
- 支撑演示页的关键统计可稳定输出；
- 基础健康检查、任务积压监控可用。

## P8 测试与交付封口
### 验收标准
- 主链路 E2E 覆盖齐全；
- API 文档清理完成；
- 部署脚本可用；
- 演示环境可启动；
- 关键业务路径有测试报告支撑。

---

# 3. 测试层次要求

## 3.1 单元测试
目标：
- 验证领域规则；
- 验证状态机；
- 验证服务方法边界条件；
- 验证加密、脱敏、权限等基础能力。

## 3.2 集成测试
目标：
- 验证 controller -> service -> repository -> db 的链路；
- 验证 Flyway 后数据库可正常使用；
- 验证事务、生效约束、日志写入。

## 3.3 端到端测试
目标：
- 验证跨模块主链路；
- 验证 AI 回调后续流程；
- 验证报告与随访闭环。

---

# 4. 必测主链路

至少覆盖以下 9 条主链路：

1. 登录；
2. 患者建档；
3. 就诊创建；
4. 病例创建；
5. 影像上传；
6. AI 分析；
7. 报告生成；
8. 医生修正；
9. 随访创建。

---

## 5.5 analysis 模块
- 创建任务成功
- 成功回调写回成功
- 失败回调写回成功
- 重复回调幂等
- 修正反馈保存成功

## 5.6 report 模块
- 生成医生版报告
- 生成患者版报告
- 版本累加
- 导出留痕

## 5.7 followup 模块
- 触发随访计划
- 创建随访任务
- 完成随访任务
- 通知记录落库

---

# 6. 国一标准下的答辩证据链要求

后端部分必须准备以下证据：

1. 架构图与模块图；
2. 数据表与主链路关系说明；
3. 一条完整病例流程演示；
4. 状态机与状态日志截图；
5. AI 任务与回调记录截图；
6. 报告版本演示；
7. 随访闭环演示；
8. 测试结果摘要；
9. 安全与权限说明。

---

# 7. 合并与发布规则

## 7.1 提交前要求
每个模块 PR 至少包含：
- 代码；
- 对应迁移或迁移校验结果；
- OpenAPI 注解；
- 单元测试；
- 集成测试或联调说明；
- 文档同步更新。

## 7.2 禁止合并的情况
- 无测试；
- 无迁移说明；
- 新增字段与字典不一致；
- 状态机绕过统一服务；
- 敏感字段明文落库；
- org_id 查询未加隔离。

---

# 8. 交付结论

本文件把“完成”定义成了可验证标准。

后续每一阶段都必须回答两个问题：

1. 功能是否存在？
2. 功能是否被测试并可演示？

只有两者都满足，才算真正交付。
