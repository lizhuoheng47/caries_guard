# 开发任务拆解清单

> 文档定位：后端执行级任务清单  
> 目标：把方案转为可排期、可分工、可验收的任务  
> 使用方法：按阶段推进，可用于项目看板、日报、周报、里程碑管理

---

# 1. 总体开发顺序

正式顺序固定为：

1. P0 基线对齐
2. P1 system
3. P2 patient + case
4. P3 image
5. P4 analysis
6. P5 report
7. P6 followup
8. P7 dashboard
9. P8 测试与交付

# 13. 结论

本清单的目标不是“列任务”，而是把项目从文档阶段推进到执行阶段。

从现在开始，项目推进应该遵循一句话：

> **所有设计都必须落成任务，所有任务都必须有验收。**
