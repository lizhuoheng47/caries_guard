# 开发任务拆解清单

## 1. 当前状态

原文档按 `P0 ~ P8` 规划任务拆解。当前代码已经完成大部分主链路能力，因此本文改为“剩余收尾任务清单”。

## 2. 已完成阶段概览

- `P1 system`：已实现登录、当前用户、系统管理接口
- `P2 patient + case`：已实现患者/就诊/病例/状态机
- `P3 image`：已实现附件、影像、质检
- `P4 analysis`：已实现任务、回调、重试、纠偏
- `P5 report`：已实现模板、生成、查询、导出审计
- `P6 followup`：已实现计划、任务、记录、自动触发
- `P7 dashboard`：已实现七个统计接口
- `P8`：已有真实数据库主链路 E2E

## 3. 当前剩余任务

### 3.1 高优先级

1. 补齐 `sys_menu` 与角色权限种子数据，形成非管理员真实演示场景
2. 明确并补全跨业务模块的数据权限规则落地范围
3. 把部署文档中的 RabbitMQ / 本地存储 / 外部依赖说明进一步产品化
4. 补齐中文 PDF 输出能力，替换当前极简 ASCII PDF 生成器

### 3.2 中优先级

1. 统一为 patient/image/report/followup 写接口补操作日志
2. 为 dashboard 增加大数据量下的缓存或宽表方案
3. 报告模板更新时递增 `version_no`
4. 为 report 导出提供真正的文件下载链路封装

### 3.3 低优先级

1. 引入 Redis 在 token/热点数据/防重中的真实用途
2. 评估 MQ 消费者与异步任务执行器放入仓库还是拆外部服务
3. 补充部门维度、医生维度 dashboard 下钻

## 4. 当前不建议再投入的方向

在现阶段，不建议继续写“更大的目标态文档”，优先级应低于：

- 种子数据
- 部署可用性
- 权限可演示性
- 报告输出质量

## 5. 协作原则

后续新增任务必须满足：

- 先确认表结构是否已有支撑
- 先确认当前文档是否已有现状描述
- 需要改表时必须走新 Flyway 迁移
- 新接口必须补齐到 `03`、`16` 等文档
