# AI 协同与回调契约规范

## 1. 当前协同边界

当前 Java 仓库承担：

- 分析任务创建与审计
- 任务状态跟踪
- AI 回调验签与写回
- 修正反馈入库
- 结果向报告/随访模块继续传递

当前仓库不承担：

- 模型推理
- 模型训练治理
- Python 侧任务消费实现

## 2. 任务创建契约

接口：

- `POST /api/v1/analysis/tasks`
- `POST /api/v1/cases/{caseId}/analysis`

当前请求的核心字段：

- `caseId`
- `patientId`
- `forceRetryFlag`
- `taskTypeCode`
- `remark`

Java 侧会组装并保存 `AiAnalysisRequestDTO`，字段包含：

- `taskNo`
- `taskTypeCode`
- `caseId`
- `patientId`
- `orgId`
- `modelVersion`
- `images[]`
- `patientProfile`

其中 `images[]` 由当前病例下 `quality_status_code = PASS` 的影像构成。

## 3. 消息发布

当前存在两种发布模式：

- `logging`：仅记录日志
- `rabbit`：通过 `RabbitTemplate` 发送消息

配置键：

```yaml
caries.analysis.messaging.mode
```

当前默认值：

- 本地 profile：`rabbit`
- E2E profile：`logging`

Rabbit 侧会发布：

- `analysis.requested`
- `analysis.completed`
- `analysis.failed`

注意：仓库中只有发布者，没有请求消费者实现。

## 4. 回调接口契约

接口：

- `POST /api/v1/internal/ai/callbacks/analysis-result`

Header 要求：

- `X-AI-Timestamp`
- `X-AI-Signature`

签名算法：

- HMAC-SHA256
- 明文：`timestamp + "." + rawBody`

时钟窗口：

- 默认允许偏差 `300` 秒

## 5. 回调状态

当前只支持：

- `PROCESSING`
- `SUCCESS`
- `FAILED`

不支持其他状态码。

## 6. SUCCESS 写回规则

AI 成功回调时，Java 侧会：

1. 更新 `ana_task_record.task_status_code = SUCCESS`
2. 写入 `ana_result_summary`
3. 提取并写入聚合列：
   - `overall_highest_severity`
   - `uncertainty_score`
   - `review_suggested_flag`
4. 替换当前任务的 `ana_visual_asset`
5. 视情况写入 `med_risk_assessment_record`
6. 将病例状态从 `ANALYZING` 迁移到 `REVIEW_PENDING`
7. 发布 `analysis.completed`

## 7. FAILED 写回规则

AI 失败回调时，Java 侧会：

1. 更新 `ana_task_record.task_status_code = FAILED`
2. 保存失败原因
3. 将病例状态从 `ANALYZING` 迁回 `QC_PENDING`
4. 发布 `analysis.failed`

## 8. 幂等与重试

当前已实现：

- 终态重复回调直接 ACK 幂等返回
- 非法状态跳转拦截
- 失败任务重试时创建新任务，不覆盖旧任务
- 新任务通过 `retry_from_task_id` 指向原失败任务
- 若旧任务已经被重试，旧任务的迟到成功回调只记录 ACK，不再写回业务数据

## 9. 医生修正反馈契约

接口：

- `POST /api/v1/analysis/corrections`
- `POST /api/v1/cases/{caseId}/corrections`

当前入库表：`ana_correction_feedback`

核心字段：

- `case_id`
- `diagnosis_id`
- `source_image_id`
- `doctor_user_id`
- `original_inference_json`
- `corrected_truth_json`
- `feedback_type_code`

## 10. 当前限制

- Java 仓库中没有 AI 拉取消息并执行推理的消费者实现
- 回调 payload 结构由 DTO 固定，兼容范围有限
- 训练集导出字段 `is_exported_for_train` 仅入库，暂无后续流程
