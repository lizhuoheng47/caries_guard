# P8 主链路 E2E 与交付收口

## 1. 当前结论

`P8` 当前代表“交付收口阶段”，而不是“尚未开始阶段”。主链路已经存在真实数据库 E2E 证据。

## 2. 当前主链路内容

`MainlineWorkflowE2ETest` 覆盖的真实链路包括：

- 登录
- 当前用户查询
- 报告模板创建
- 患者建档
- 患者敏感字段加密/脱敏校验
- 就诊创建
- 病例创建
- 文件上传
- 影像建档
- 影像质检
- AI 分析任务创建
- AI 成功回调
- 医生修正反馈
- 报告生成
- 报告导出审计
- 自动创建随访计划和任务
- 随访记录录入
- dashboard 总览与随访统计校验

## 3. 当前可作为交付证据的点

- 状态机不是文档规则，而是可被测试断言的真实逻辑
- 关键表都有真实写入：`med_case_status_log`、`ana_*`、`rpt_*`、`fup_*`
- 登录、导出、随访通知均有日志/审计表支撑
- dashboard 数据不是写死，而是来自业务表聚合

## 4. 当前仍未收口的工程项

- 普通角色权限种子数据
- 生产级 PDF 输出
- 外部 AI 服务真实联机
- 对象存储从本地文件系统切到真实云存储/MinIO

## 5. 对外口径

向评审说明时，建议使用以下说法：

- 后端主链路已具备真实数据库 E2E 证据
- 当前仍有少量工程化缺口，但业务闭环已经跑通
