# 文档说明与项目总览

更新日期：2026-04-15

本目录只保留当前项目最终参考文档，不保留开发过程记录。内容以 `backend-java` 当前代码、Flyway V017 和已实现功能为准。

## 1. 当前结论

1. Java 后端已实现患者、就诊、病例、影像、AI 分析、医生修正、报告、随访、看板、系统权限的业务闭环。
2. MinIO 已作为当前唯一长期对象存储口径落地，统一配置前缀为 `caries.storage`。
3. 对象存储基础设施位于 `caries-integration`：`ObjectStorageClient`、`MinioObjectStorageClient`、`StorageProperties`、`ObjectStorageAutoConfig`、`ObjectKeyGenerator`。
4. 业务层仍通过 `caries-image` 的 `ObjectStorageService` 端口访问对象存储，运行时实现由 `caries-integration` 适配提供。
5. 四个固定 bucket：`caries-image`、`caries-visual`、`caries-report`、`caries-export`。
6. AI 请求载荷 `AiAnalysisRequestDTO` 包含 `accessUrl`、`accessExpireAt`、`storageProviderCode`、`attachmentMd5`，`localStoragePath` 当前 MinIO 模式下固定为空。
7. AI 回调契约为 `AiAnalysisResultCallbackCommand`，`visualAssets` 支持 `attachmentId` 或 `bucketName + objectKey` 两种方式。
8. V014 已新增模型治理、任务 trace/耗时、修正反馈训练准入、角色菜单权限种子；V015 增加 visual asset trace 字段，V016 固化 attachment object key 治理，V017 冻结 analysis 数据字典 v2。
9. 报告 PDF 生成后写入 `caries-report`；报告导出会复制为 `caries-export` 附件并返回 `attachmentId`、`downloadUrl`、`expireAt`。
10. `Risk` 是风险评估能力，不是独立模块；`ModelAdmin` 保留治理原则，但当前未落地为独立业务模块。

## 3. 项目模块

| Maven 模块 | 作用 |
| --- | --- |
| `caries-boot` | 启动类、配置、Flyway |
| `caries-common` | 通用响应、异常、分页 |
| `caries-framework` | 权限、安全上下文、Trace、操作日志 |
| `caries-system` | 登录、用户、角色、菜单、字典、配置、数据权限 |
| `caries-patient` | 患者、就诊、病例、牙位、诊断 |
| `caries-image` | 附件元数据、影像、质检、对象存储业务端口 |
| `caries-analysis` | AI 分析任务、回调、结果、风险、修正反馈 |
| `caries-report` | 模板、报告、PDF、导出 |
| `caries-followup` | 随访计划、任务、记录 |
| `caries-dashboard` | 业务看板、模型运行质量看板 |
| `caries-integration` | MinIO 对象存储和外部集成边界 |
