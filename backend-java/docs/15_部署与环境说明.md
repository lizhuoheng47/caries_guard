# 部署与环境说明

## 1. 文档目标

本文件用于说明当前阶段后端如何在本地或演示环境启动，以及各环境的真实依赖口径。

当前结论：

- 项目可在单机环境启动
- 当前本地环境已接入真实 RabbitMQ 事件发布
- 当前对象存储默认走本地文件系统实现

## 2. 环境要求

基础依赖：

- JDK 17
- Maven 3.9+
- MySQL 8.x

当前阶段可选依赖：

- Redis：建议可用，但当前主链路不是强依赖
- RabbitMQ：`local` profile 已启用真实 publisher；Python AI 消费端与内部消费者按后续集成阶段接入

## 3. 配置文件

主要配置文件：

- `caries-boot/src/main/resources/application.yml`
- `caries-boot/src/main/resources/application-local.yml`
- `caries-boot/src/test/resources/application-e2e.yml`

当前 profile：

- 默认运行 profile：`local`
- 真库测试 profile：`e2e`

## 4. 本地运行配置

### 4.1 数据库

`application-local.yml` 当前默认值：

- url: `jdbc:mysql://127.0.0.1:3306/cg?useUnicode=true&characterEncoding=UTF-8&serverTimezone=UTC`
- username: `root`
- password: `1234`

说明：

- Flyway 默认开启
- 启动前确保数据库 `cg` 可用

### 4.2 JWT 与敏感字段

当前本地支持以下环境变量覆盖：

- `CARIES_JWT_SECRET`
- `CARIES_JWT_EXPIRE_SECONDS`
- `CARIES_CRYPTO_SECRET`
- `CARIES_HASH_SECRET`
- `CARIES_IMAGE_ACCESS_SECRET`
- `CARIES_ANALYSIS_CALLBACK_SECRET`
- `CARIES_RABBIT_HOST`
- `CARIES_RABBIT_PORT`
- `CARIES_RABBIT_USERNAME`
- `CARIES_RABBIT_PASSWORD`
- `CARIES_ANALYSIS_EXCHANGE`
- `CARIES_ANALYSIS_REQUESTED_QUEUE`
- `CARIES_ANALYSIS_COMPLETED_QUEUE`
- `CARIES_ANALYSIS_FAILED_QUEUE`
- `CARIES_ANALYSIS_REQUESTED_ROUTING_KEY`
- `CARIES_ANALYSIS_COMPLETED_ROUTING_KEY`
- `CARIES_ANALYSIS_FAILED_ROUTING_KEY`

建议：

- 本地开发可沿用默认值
- 演示环境与正式环境必须替换默认 secret

### 4.3 文件存储

当前默认实现类：

- `LocalObjectStorageService`

当前配置：

- local root: `${user.dir}/var/image-storage`
- bucket: `caries-image`

说明：

- 当前不会强制要求 MinIO
- 报告 PDF、上传影像、附件访问均可走本地文件系统

### 4.4 RabbitMQ

`application-local.yml` 当前默认值：

- host: `${CARIES_RABBIT_HOST:127.0.0.1}`
- port: `${CARIES_RABBIT_PORT:5672}`
- username: `${CARIES_RABBIT_USERNAME:guest}`
- password: `${CARIES_RABBIT_PASSWORD:guest}`
- messaging mode: `rabbit`

当前声明的 Rabbit 资源：

- exchange: `caries.analysis.exchange`
- queue: `caries.analysis.requested.queue`
- queue: `caries.analysis.completed.queue`
- queue: `caries.analysis.failed.queue`
- routing key: `analysis.requested`
- routing key: `analysis.completed`
- routing key: `analysis.failed`

说明：

- Java 创建分析任务后，会把 AI 请求 JSON 发布到 `analysis.requested`
- Java 处理回调成功/失败后，会继续发布 `analysis.completed` / `analysis.failed`
- `e2e` profile 仍使用 `logging` 模式，避免测试依赖本地 MQ 安装状态

## 5. 启动方式

在仓库根目录执行：

```bash
mvn -pl caries-boot -am spring-boot:run
```

默认端口：

- `8080`

启动后可检查：

- `GET /api/v1/system/ping`
- `GET /swagger-ui.html`
- `GET /v3/api-docs`

## 6. 初始账号

由 Flyway 初始化：

- username: `admin`
- password: `123456`

用途：

- 本地开发
- 演示环境联调
- boot 真库测试默认管理员

## 7. 测试环境

`application-e2e.yml` 当前配置：

- database: `cg_e2e`
- username: `${CARIES_E2E_DB_USER:root}`
- password: `${CARIES_E2E_DB_PASS:1234}`
- `createDatabaseIfNotExist=true`

说明：

- 用于 boot 真库测试
- 避免污染本地开发库 `cg`

## 8. 当前推荐验证命令

### 8.1 编译与测试

```bash
mvn test
```

### 8.2 关键真库链路

```bash
mvn -q -pl caries-boot -am "-Dtest=MainlineWorkflowE2ETest,AnalysisToReportE2ETest,AnalysisReportFollowupE2ETest,FollowupTriggerIdempotencyE2ETest,DashboardOverviewIntegrationTest,DashboardTrendIntegrationTest" "-Dsurefire.failIfNoSpecifiedTests=false" test
```

## 9. 当前部署边界声明

- 当前已切真实 RabbitMQ publisher，但仓库内未提供 Python AI 消费者实现
- 当前 `analysis.completed` / `analysis.failed` 仅负责发布，未在本仓库内接入真实消费处理链
- 当前对象存储默认走本地文件系统，不是生产对象存储方案
- 当前没有提供 Docker Compose 或一键部署脚本
- 当前演示环境建议按“单机 Java + 单机 MySQL”部署
