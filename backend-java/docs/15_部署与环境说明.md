# 部署与环境说明

## 1. 当前环境结论

本文按当前代码与配置重写，重点说明“真实依赖是什么”。

## 2. 本地默认配置

文件：

- `caries-boot/src/main/resources/application.yml`
- `caries-boot/src/main/resources/application-local.yml`

默认参数：

- server port: `8080`
- MySQL database: `cg`
- MySQL user/password: `root / 1234`
- JWT secret: 可通过 `CARIES_JWT_SECRET` 覆盖
- 本地对象存储目录：`${user.dir}/var/image-storage`

## 3. E2E 测试配置

文件：

- `caries-boot/src/test/resources/application-e2e.yml`

特点：

- 数据库：`cg_e2e`
- Flyway：允许 clean
- 对象存储目录：`${user.dir}/var/image-storage-e2e`
- analysis messaging mode：`logging`

## 4. 当前真实依赖

### 4.1 必需依赖

- JDK 17
- Maven 3.9+
- MySQL 8.x

### 4.2 条件依赖

- RabbitMQ：本地 profile 下如果要创建分析任务，需要可用 RabbitMQ
- Redis：当前业务主链路不依赖 Redis，缺失时一般不影响主要 API 功能

## 5. 关于 RabbitMQ 的真实口径

当前代码存在两种分析消息发布模式：

- `rabbit`
- `logging`

本地 `application-local.yml` 当前配置为：

```yaml
caries.analysis.messaging.mode: rabbit
```

因此：

- 应用本身通常可以启动
- 但执行“创建分析任务”时会尝试向 RabbitMQ 发消息
- 如果 RabbitMQ 不可用，分析任务创建链路会受影响

## 6. 关于对象存储的真实口径

配置项命名保留了对象存储抽象：

- `bucket-name`
- `provider-code`
- `local-root`

但当前实现类是：

- `LocalObjectStorageService`

所以当前真实落盘位置是本地文件系统，不是 MinIO/S3/OSS。

## 7. 启动命令

```bash
mvn -pl caries-boot -am spring-boot:run
```

## 8. 访问地址

- `http://127.0.0.1:8080/api/v1/system/ping`
- `http://127.0.0.1:8080/v3/api-docs`
- `http://127.0.0.1:8080/swagger-ui.html`

## 9. 当前库内容说明

默认 `cg` 库不是业务样例库，而是系统种子库：

- 有管理员、角色、字典
- 没有患者/病例/报告/随访样例数据

如需演示完整链路，应使用 API 自行创建业务数据或直接运行 E2E 测试链路。

## 10. 部署前必须修改项

- JWT secret
- sensitive crypto/hash secret
- image access secret
- analysis callback secret
- MySQL 账号密码
- RabbitMQ 账号密码
