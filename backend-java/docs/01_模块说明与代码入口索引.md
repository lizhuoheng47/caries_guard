# 01 模块说明与代码入口索引

## 1. 模块边界

Java 模块按业务域划分：

- `caries-system`：用户、角色、组织、权限；
- `caries-patient`：患者、就诊、病例；
- `caries-image`：影像和附件；
- `caries-analysis`：AI analysis task、callback、summary、visual asset；
- `caries-report`：报告；
- `caries-followup`：随访；
- `caries-integration`：外部服务集成；
- `caries-boot`：启动入口。

## 2. Analysis 入口

关键职责：

- 创建 analysis task；
- 发布 MQ；
- 接收 Python callback；
- 保存 `ana_result_summary.raw_result_json`；
- 保存 visual asset；
- 按 `needsReview` 推进复核。

## 3. RAG 入口

Java 提供鉴权、上下文组装和展示入口。Python 完成知识检索和通用大模型生成。

Java 不实现知识向量检索，不实现大模型微调。

## 4. 代码定位建议

| 职责 | 建议层 |
| --- | --- |
| Controller | API 入参和鉴权 |
| AppService | 业务编排 |
| DomainService | 业务规则 |
| Repository/Mapper | 数据访问 |
| IntegrationClient | Python/RAG/MinIO/MQ 调用 |

## 5. AI 字段处理

强类型字段用于稳定摘要，扩展字段放入 `raw_result_json`。

必须兼容：

- `gradingMode`
- `gradingImplType`
- `uncertaintyMode`
- `uncertaintyImplType`
- `needsReview`
